SYNOPSIS

  • field pathName ?option value...?
  •     -changecmd command
  •     -state state
  • pathName cget option
  • pathName configure option value...

  • pathName set value

DESCRIPTION

A map field(i) is a widget that allows a data item to be entered in a dialog box. marsgui(n) provides a number of field types; all of them implement at least the interface described here.

Commands

Each field(i) widget type provides at least the following command:

field pathName ?option value...?

As usual, it creates an instance of the type called pathName, given the specified options, and returns the pathName.

Every field(i) widget supports at least the following options:

-changecmd command
Specifies a command to be called whenever the field's value changes, for any reason whatsoever (including explicit calls to set. The new value is appended to the command as an argument.
-state state
field(i) widgets have states, just as the normal tk:: and ttk:: widgets do. The valid states are normal and disabled.

Instance Command

Every field(i) widget provides at least the following instance commands:

pathName cget option
Returns the value of the specified option.
pathName configure option value...
Sets the value of one or more of the object's options.
pathName get
Returns the field's current value.
pathName set value

Explicitly sets the field's value, and calls the -changecmd if the value has really changed, i.e., successive calls to set with the same value will result in at most one call to the -changecmd.

To clear the field, set its value to "".
