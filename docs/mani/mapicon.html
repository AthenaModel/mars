mapicon(i) -- Mapcanvas Icon Type Interface

SYNOPSIS

  • mapicon name canvas cx cy ?options?
  • mapicon typename
  • obj cget option
  • obj configure option value...
  • obj draw cx cy

DESCRIPTION

A mapicon(i) is snit::type that implements a mapcanvas(n) icon type. Mapcanvas icons are graphics symbols, possibly composed of multiple canvas(n) items than can be positioned and manipulated using map coordinates and reference strings via the mapcanvas(n) icon command.

A mapicon(i)'s primary responsibility is to draw the icon on the canvas to begin with; then, various aspects of the icon's appearance can be manipulated using the icon's options (if any).

When the object is destroyed, it should remove itself from the canvas, using catch(n) in case the canvas has already been destroyed.

CANVAS TAGS

Every icon should be tagged with the following canvas tags, in the following order:

  • Its id. This is the final component of its command name.
  • Any tags required to identify pieces of the icon to support changes to option values. These should be called names like "id.this" and "id.that".
  • Its typename.
  • The tag "icon".
  • Any additional tags, as specified by -tags.

OPTIONS

Every mapicon(i) should define at least the following options:

-tags taglist
This is a list of additional canvas tags that should be applied to this particular icon.

COMMANDS

Every mapicon(i) module must define a command like the following:

mapicon name canvas cx cy ?options?

Creates a new mapicon(i) object called name. The object is represented as a new Tcl command in the caller's scope; mapicon returns the fully-qualified form of the name.

The canvas is the window name of the mapcanvas(n) on which the icon will appear. It will be drawn at canvas coordinates cx,cy.

The options vary by icon type. The standard Snit info options method may be used to retrieve the option names.

mapicon typename
Returns the name by which the icon type will be known to users of the mapcanvas(n) widget.

INSTANCE COMMAND

Each instance of a mapicon object has the following subcommands:

obj cget option
Returns the value of the specified option.
obj configure option value...
Sets the value of one or more of the object's options.
obj draw cx cy
Redraws the icon at the specified coordinates, using the current option settings. The previous instance of the icon (if any) is deleted from the canvas.

ENVIRONMENT

Requires Tcl 8.5 or later.

To use this package in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
