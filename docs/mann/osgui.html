osgui(n) -- Platform-Dependent GUI Utilities, submodule of marsgui(n)

SYNOPSIS

package require marsgui 3.0.23
namespace import ::marsgui::*
  • osgui mktoolwindow w parent

DESCRIPTION

osgui(n) provides platform-independent GUI routines for features that must be implemented differently on different windowing systems. Ideally, all windowing system code should reside here or in global.tcl, the global appearance module.

COMMANDS

This module defines the following commands:

osgui mktoolwindow w parent

Makes window w a toolwindow over its parent window. That is:

  • Window w is declared transient over parent. In most cases, this is sufficient.
  • On MS Windows, special handling is required:
    • Window w is given the -toolwindow wm attribute.
    • Window w is positioned near the upper left of the parent window.

Windows w and parent will usually be toplevels, but osgui mktoolwindow accepts children of the respective toplevels as well.

ENVIRONMENT

This package requires Tcl/Tk 8.5 or later.

To use this package in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
