DESCRIPTION

A logreader(n) object reads and parses a log file given a user-defined parsing function. There are two ways to use a logreader:

  • The get method returns the parsed contents of an entire log file given the file name.
  • The newentries method allows the application to periodically retrieve new entries from an active log file.

logreader(n) places few requirements on either the log file or the parsing function. It assumes that:

  • Log file entries are written atomically--i.e., that it need not worry about reading a partial entry from the disk.
  • A newline character at the end of the file can be ignored.

A log file with newlines between entries will meet these requirements, provided that it is output as a line-buffered file: 

    set channel [open $filename w]
    fconfigure $channel -buffering line

COMMANDS

This module defines the following command:

logreader pathName ?options...?

Creates a new logreader component named pathName.

The logreader widget may be created with the following option:

-parsecmd cmd
A parsing command used to parse the contents of a log file into the proper application specific format. It will be passed one argument, a chunk of zero or more entries from the log file. The input may be parsed in any way desired by the application. Required.

INSTANCE COMMAND

$logreader get file
Returns the parsed content of the named file, as parsed by the -parsecmd.
$logreader newentries file
Given a new file name, reads all entries from the file and returns the parsed content, leaving the file open. Subsequent calls parse and return any new entries; if there are none, -parsecmd will be called on the empty string. When newentries is called on a different file, the current file is closed and the new file is opened.
$logreader close
Explicitly closes the current log file, if any.
$logreader filename
Returns the name of the most recently read log file, as it was passed to get or newentries.
$logreader isfileopen
Returns 1 if the current file is open, else 0.

ENVIRONMENT

marsutil(n) requires Tcl 8.5 or later.

To use marsutil(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
