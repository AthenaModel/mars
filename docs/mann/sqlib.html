DESCRIPTION

sqlib(n) defines a number of utility commands for use with SQLite database files.

SQLite databases are accessed through a command created via the sqlite3 command; the sqlib(n) commands take such a database access command as their first argument.

COMMANDS

sqlib clear db
Deletes all persistent and temporary schema elements (tables, indices, views, triggers) from the database with the fully qualified handle db. SQLite3 special tables (e.g., sqlite_sequence) cannot usually be dropped, but all entries are deleted. Databases attached to db using the "DATABASE ATTACH" SQL statement are ignored.
sqlib saveas db filename

Saves a copy of the persistent contents of the database with the fully qualified handle db. The copy will be saved as a new database file with the specified filename. It's an error if a file with that name already exists.

Temporary tables and databases attached to db using the "DATABASE ATTACH" SQL statement are ignored.

Warning: if the db is in the middle of a transaction, the transaction will be committed before the copy is saved, and a new transaction will begin.

sqlib compare db1 db2

Compares the persistent contents of db1 and db2, both of which should be fully-qualified database handles. First the schemas are compared, followed by the data in the tables. (Note that this can be a lengthy process if the databases are large.) Returns a string describing the first difference found, or the empty string if no differences are found.

Temporary tables and databases attached to db using the "DATABASE ATTACH" SQL statement are ignored.

sqlib tables db
Returns a list of the names of the tables defined in the SQLite database with the fully-qualified handle db. The list includes all accessible tables, including SQLite3 internal tables (except for sqlite_master and sqlite_temp_master), temporary tables, and tables in databases attached to db using the "DATABASE ATTACH" SQL statement.
sqlib schema db ?table?

Returns the SQL CREATE statements used to define all tables, views, and indices in database db. If table is specified, it may be a table/view/index name, or a glob pattern; in this case the output is restricted to those tables, views, and indices whose names match table.

The output includes all persistent and temporary tables, indices, views, and triggers in the database. Entities in databases attached to db using the "DATABASE ATTACH" SQL statement are ignored.

sqlib query db sql ?options...?

Performs an arbitrary sql query on database db, formatting any results appropriately for display. Newlines in data values are escaped, and long text strings are truncated.

sql can contain any SQL statements understood by SQLite, but in practice there's no reason to use this command except for SELECT queries.

Note: as of Mars 2.17, the sql query can reference variables in the caller's context, as it can for the normal SQLite3 eval subcommand.

The command supports the following options:

-mode mc|list|csv|json

Specifies the output mode; the default is mc, multicolumn.

When -mode is mc, each record is output on a single line. Fields are formatted in columns, with the field name as the column header.

When -mode is list, the output is a list of records. Each record is output on multiple lines, with (nominally) one line per field. Each line consists of the field name and value; the record's field names are values are formatted neatly in parallel columns.

When -mode is csv, the output is in CSV (comma-separated value) format, one record per line, beginning with a header record of column labels. Non-numeric values are enclosed in double-quotes.

When -mode is json, the output is a set of JSON objects, one per record. Each object contains the name/value pairs of a row from the database. The output is properly formatted with commas between each object and the entirety of the output enclosed in brackets ([ and ]).

-channel chan
When this option is supplied, the output is directed to the specified channel. The channel can be a file, a socket or stdout. Note that if the -filename option is also supplied, it overrides this option.
-filename name
When this option is supplied, the output of the query is directed to the file specified. It is an error if the file cannot be opened or written to. This option will override the -channel option if it is also specified.
-labels list
Normally sqlib query uses the column names found in the query to label the fields in the output. If specified, the value of -labels is a list of labels to be used instead. Note: This option is ignored when -mode is json.
-maxcolwidth chars

Sets the maximum column width, in characters, when -mode is mc; the default is 30. All data values longer than this will be truncated with "...".

If -maxcolwidth is set to 0, values will not be truncated; this is often a Bad Thing, as it can result in extremely wide output.

-headercols num
Sets the number of header columns. When -mode is mc, repeated values in a column are skipped. That is, if a column contains identical values in consecutive rows, the column will be blank in all but the first of the rows. Defaults to "0".
sqlib mat db table iname jname ename ?options...?

Extracts a mat(n) matrix from a table in database db. The values in the table's iname column define the rows of the matrix, the values in the table's jname column define the columns, and the ename column provides the elements of the matrix.

The command supports the following options:

-ikeys keys

keys is a list of the values from the iname column which should be included in the matrix; the rows of the matrix will have the same order as the keys. keys may be a subset of the values which appear in that column.

If keys is not given, each unique value in the iname column will have a row in the matrix, in order by first appearance.

-jkeys keys

keys is a list of the values from the jname column which should be included in the matrix; the columns of the matrix will have the same order as the keys. keys may be a subset of the values which appear in that column.

If keys is not given, each unique value in the jname column will have a column in the matrix, in order by first appearance.

-defvalue value
The matrix will be initialized with the specified value, or with the empty string if -defvalue is omitted.
-returnkeys flag
Normally, the command returns the desired matrix. If the flag is true, then the command will return a list of three elements: the matrix, the list of row keys, and the list of column keys. This option is usually used when -ikeys or -jkeys is omitted, since they order of the rows and columns in the matrix might otherwise be obscure.
sqlib insert db table dict

Inserts the contents of dictionary dict into the db as a row in the named table. The dictionary keys must be the same as the column names in the table. It's an error if a row already exists with the same key column values.

This will be less efficient than an explicit "INSERT INTO" with hardcoded column names, but where performance isn't an issue it wins on maintainability.

Compare to sqlib replace.

sqlib replace db table dict

Inserts the contents of dictionary dict into the db as a row in the named table. The dictionary keys must be the same as the column names in the table. If there's already a row with matching key columns, it will be replaced.

This will be less efficient than an explicit "INSERT OR REPLACE INTO" with hardcoded column names, but where performance isn't an issue it wins on maintainability.

Compare to sqlib insert.

sqlib grab ?-insert? db table condition ?table condition...?

Grabs a collection of rows from one or more tables in the database, and returns them to the user as one value. The grabbed data can later be restored to the database using sqlib ungrab. For example, an operation that updates rows in the database might grab the rows prior to the change so that the original values can be restored afterwards.

The returned value is called a "grab data set". It is a flat list structure {tableSpec values tableSpec values...} where tableSpec identifies the table and values is a flat list of column values for all of the grabbed rows from the table. As returned by sqlib grab, the tableSpec is simply the name of the table; if the optional -insert option is included, however, it will be a list {tableName INSERT}. The INSERT tag tells sqlib ungrab to use INSERT to put the data in the table rather than UPDATE.

NULL values are retrieved as the SQLite3 "nullvalue", which defaults to the empty string.

For example, 

    set data [sqlib grab $db table1 {x=5} table2 {y=10}]

will grab all rows from table1 where x = 5, and all rows from table2 where y = 10.

The result of calling sqlib grab can be passed to the sqlib ungrab command to put the rows back in the database.

sqlib ungrab db data

Given a "grab data set" produced by sqlib grab or some other command, this command puts the data back into the database using either UPDATE or INSERT, as determined by the optional INSERT tag in each table's table specification. For example, the following code grabs rows from the database, modifies them, and then restores the original values. 

    set data [sqlib grab $db table1 {x=5} table2 {y=10}]
    ...
    # Code that updates the grabbed rows.
    ...
    sqlib ungrab $db $data

If the -insert tag is included in the sqlib grab call, this command can also be used to re-insert deleted rows.

Note that the schemas of the relevant tables should not be altered between grabbing and ungrabbing, or the ungrabbing will have unpredictable results.

Whether updating or inserting, values that match the SQLite3 "nullvalue" will be inserted as NULL. When updating, however, it is assumed that key columns do not contain NULL.

sqlib fklist db table ?-indirect?
Returns a list of the tables with foreign keys that reference the specified table, excluding table itself. If the -indirect option is included, the list also includes the tables that depend indirectly on table. It is an error if table is unknown.

ENVIRONMENT

sqlib(n) requires Tcl 8.5 or later, and SQLite 3.2 or later.

To use sqlib(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the marsutil(n) library directory.
