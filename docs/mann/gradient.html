DESCRIPTION

The gradient(n) module defines a type for defining and computing color gradients: shadings of color used to visualize a range of input levels.

A gradient is defined by six parameters: the input range, specified as a minimum level, a midpoint, and a maximum level, and the colors associated with each of these points, defined in #rrggbb form. Then, given any numeric value between the minimum and maximum levels, the gradient will output a color in #rrggbb form. For inputs between the minimum level and the midpoint it is scaled between the minimum color and the midpoint color; for inputs between the midpoint and the maximum level, it is scaled between the midpoint color and the maximum color. For example, a value between -100 and +100 could gradate from red at -100 to white at 0 to green at +100.

COMMANDS

This module defines the following command:

gradient pathName ?options...?

Creates a new gradient widget named pathName.

The gradient widget may be created with the following options; the defaults cause inputs from 0.0 to 1.0 to produce shades of gray from white to black.

-mincolor color
Specifies the color, in #rrggbb form, which corresponds to the minimum input level, -minlevel. Defaults to #FFFFFF.
-midcolor color
Specifies the color, in #rrggbb form, which corresponds to the midpoint input level, -midlevel. Defaults to #FFFFFF.
-maxcolor color
Specifies the color, in #rrggbb form, which corresponds to the maximum input level, -maxlevel. Defaults to #000000.
-minlevel level
Specifies the minimum input level. Defaults to 0.0.
-midlevel level
Specifies the midpoint input level. Defaults to 0.0.
-maxlevel level
Specifies the maximum input level. Defaults to 1.0.

INSTANCE COMMAND

$gradient configure option value...
Sets the values of the specified options.
$gradient cget option
Returns the value of the specified option.
$gradient color level
Given an input level ranging from -minlevel to -maxlevel, returns an output color (in #rrggbb form) scaled from -mincolor to -midcolor to -maxcolor.

ENVIRONMENT

marsutil(n) requires Tcl 8.5 or later.

To use marsutil(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
