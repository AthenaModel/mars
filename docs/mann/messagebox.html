DESCRIPTION

This module defines a replacement for the tk_messageBox(n) command that offers the following enhancements:

  • The icons are slightly improved.
  • The button constants and label text can be set explicitly.
  • The fonts are more in keeping with the theme.
  • If desired, the dialog will include a "Do not show this message again" checkbox. See Ignoring Messages for more information.
  • If desired, the user can enter a text string, select an item from a list, or select multiple items from a list, in addition to just selecting buttons.

Ignoring Messages

messagebox(n) allows the application to define message dialogs that the user can choose to ignore, at least until the application is restarted. The process is as follows:

  • The application passes an -ignoretag that uniquely identifies this message.
  • As a result, the dialog will contain a check box labeled "Do not show this message again".
  • If the user checks the check box, messagebox(n) will remember that this message should be ignored in the future.
  • When the application requests the message again, messagebox(n) will simply return the -ignoredefault value.
  • The ignore flags can be cleared using the reset subcommand.

COMMANDS

This module defines the following command:

messagebox popup ?options...?

Pops up the message box, given the following options. The command returns the symbolic name of the selected button.

-buttons dict
A dictionary of symbolic constants and label strings that defines the set of buttons to appear at the bottom of the message box. Defaults to {ok "OK"}.
-default symbol
Specifies the symbolic constant of the default button. Defaults to the symbolic constant of the first button in -buttons.
-icon icon
Specifies which icon should be included in the message box: error, info, question, warning, or peabody.
-ignoretag tag
If given, the message box will include a "Do not show this message again" check box. See Ignoring Messages for more information.
-ignoredefault symbol
Specifies the symbolic constant to return when user has previously asked to ignore this message. Defaults to the value of -default.
-message text
The message text to display. The text will be wrapped automatically; newlines are honored.
-onclose symbol
Specifies the symbolic constant associated with the window manager's "close" button. Defaults to the symbolic constant of the first button in -buttons.
-parent window
The message box will appear over the specfied window.
-title string
The dialog window will have the specified title string.
-wraplength len
Species the -wraplength for the Tk label widget that will display the user's message. Defaults to "3i". If "" is given, no wrapping will be done.
messagebox reset ?tag?
Clears the ignore flag for the specified ignore tag. If no tag is given, clears all ignore flags.
messagebox gets ?options...?

Pops up a message box containing a text entry widget, given the following options. On "Cancel", returns the empty string; on "OK", validates and returns the entered text, trimmed of leading and trailing whitespace. Pressing the window manager's close button is the same as pressing "Cancel".

-oktext text
Specifies the text for the OK button. Defaults to OK.
-icon icon
Specifies which icon should be included in the message box: error, info, question, warning, or peabody. Defaults to question
-initvalue text
Specifies an initial value for the entry widget. If text is not the empty string, the widget will contain the text, already selected. Otherwise, the widget will be empty.
-message text
The message text to display. The text will be wrapped automatically; newlines are honored.
-parent window
The message box will appear over the specfied window.
-title string
The dialog window will have the specified title string.
-validatecmd command
Specifies a validation command. The command should take one additional argument, the string to validate. If the string is valid, it should be returned; the validation command can canonicalize it if desired (e.g., convert it to uppercase). If the string is invalid, the command should throw an error with error code INVALID and a human-readable error message. The error message will be displayed below the entry widget.
messagebox pick ?options...?

Pops up a message box containing a menu box, given the following options. On "Cancel", returns the empty string; on "OK", returns the item selected from the menu box's pulldown. Pressing the window manager's close button is the same as pressing "Cancel".

-oktext text
Specifies the text for the OK button. Defaults to OK.
-icon icon
Specifies which icon should be included in the message box: error, info, question, warning, or peabody. Defaults to question
-initvalue text
Specifies an initial value for the menu box.
-message text
The message text to display. The text will be wrapped automatically; newlines are honored.
-parent window
The message box will appear over the specfied window.
-title string
The dialog window will have the specified title string.
-values list
Specifies the list of values for the menu box's pulldown.
messagebox listselect ?options...?

Pops up a message box containing an listfield(n) widget, given the following options, allowing the user to select multiple items from a list. The list is displayed in parallel columns; the right-hand column shows the items the user has chosen, and the left-hand column shows all remaining items.

On "OK", returns a list whose first element is ok and whose second element is a list of the keys of the items selected from the list field. On "Cancel", returns a list of one element, cancel. Pressing the window manager's close button is the same as pressing "Cancel".

-oktext text
Specifies the text for the OK button. Defaults to OK.
-icon icon
Specifies which icon should be included in the message box: error, info, question, warning, or peabody. Defaults to question
-initvalue text
Specifies an initial value for the list field, a list of keys from the -itemdict.
-message text
The message text to display. The text will be wrapped automatically; newlines are honored.
-parent window
The message box will appear over the specfied window.
-title string
The dialog window will have the specified title string.
-itemdict dict
Specifies the item dictionary that the user will choose from. The values are displayed for the user; the call will return the keys of the selected items.
-showkeys flag
If this option is true, the values the displayed to the user will be prefixed by the keys. Defaults to false.
-stripe flag
If this option is true, even-numbered rows will be striped with a distinctive color. This is useful if the row values might wrap to multiple lines.
-listrows rows
The list items are displayed in scrolling lists. This option controls the height of these lists in rows of text.
-listwidth chars
The list items are displayed in scrolling lists. This option controls the width of these lists in characters.

HISTORY

The original package was written for Athena, and then moved to Mars for general use. The icons are based on the appearance of the tk_messageBox(n) icons.

The gets subcommand was added on 4/19/2010.

The pick subcommand was added on 1/28/2011.

The listselect subcommand was added on 9/29/2011.
