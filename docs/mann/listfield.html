  •     -stripe flag
  •     -width chars

  • pathName configure option value...

  • pathName set value

DESCRIPTION

The listfield(n) widget is field(i) data entry field for selecting multiple entries from a list of choices. The available choices are shown in two columns; arrow buttons are used to move items from the "omitted" list to the "included" list.

The choices are specified by setting the -itemdict option. The value of the widget is a list of the dictionary keys of the items in the "included" list. The dictionary values are displayed in the two item columns, optionally prefixed by the keys.

COMMANDS

This module defines the following commands.

listfield pathName ?options...?

Creates a new listfield widget named pathName, and returns pathName. The widget supports the following options:

-changecmd command
From field(i). Specifies a command to be called whenever the field's value changes, for any reason whatsoever (including explicit calls to set. The new value is appended to the command as an argument.
-height rows
The height of the "include" and "omit" lists, in rows.
-itemdict dict
The items from which the user can choose, specified as a dictionary of keys and values. The dictionary values are displayed in the "include" and "omit" lists, optionally prefixed by the keys (see -showkeys); the get and set methods return and accept a list of keys.
-showkeys flag
If true (the default), the text shown for each item in the item lists is "key: value". Otherwise only the -itemdict values are displayed.
-state state
From field(i). listfield(i) widgets have states, just as the normal tk:: and ttk:: widgets do. The valid states are normal and disabled.
-stripe flag
If true (the default) the even items in each list are striped with a distinctive background color. This is useful if item values might wrap to multiple lines.
-width chars
The width in text characters of each of the item lists.

INSTANCE COMMAND

listfield(n) widgets have the following subcommands:

pathName cget option
From field(i). Returns the value of the specified option.
pathName configure option value...
From field(i). Sets the value of one or more of the object's options.
pathName get
From field(i). Returns the field's current value.
pathName set value

From field(i). Explicitly sets the field's value, and calls the -changecmd if the value has really changed, i.e., successive calls to set with the same value will result in at most one call to the -changecmd.

If the value contains an element that isn't a valid -itemdict key, that element is ignored. This allows the field to be used with data values that have become invalid due to changes elsewhere in the system.

ENVIRONMENT

marsgui(n) requires Tcl 8.5 or later.

To use marsgui(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
