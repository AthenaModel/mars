DESCRIPTION

The cmsheet(n) widget is a Tktable mapped to a cellmodel(n) model. The client maps particular cellmodel(n) cells to rows, columns, and regions in the Tktable; thereafter, edits in editable cells will cause the cell model to recompute and display the updated information.

For the most part this man page stands alone; however, it will be useful to review the Tktable documentation, with particular reference to cell indices, tags, and embedded windows.

Mapping Cells

To use cmsheet(n), first define a cellmodel(5) model, and load it into an instance of cellmodel(n). Next, create an instance of cmsheet(n), and link it to the cell model using the -cellmodel option. Finally, use mapcell and its peers to map particular cell model cells into the sheet's table layout. Constant cells will be editable by default; formula cells will not.

Note that cells can be re-mapped dynamically. For example, consider a parameter that can be computed by the application or entered by the user. The cell model can have two cells, one a formula and one a constant, and the application can map one or the other as desired.

OPTIONS

The widget has the following options.

-browsecommand cmd
Specifies a command to be called when the user browses to a different cell in the cmsheet(n). See the TkTable documentation for more details, this option is delegated to it.
-cellmodel cmd
The name of an instance of cellmodel(n). Note that multiple cmsheet(n) widgets can be mapped to a single instance of cellmodel(n).
-changecmd cmd

Specifies a command to be called whenever a cell is successfully edited. The cmd must be a command prefix taking two additional arguments, the name of the cell in the cellmodel(n) and the new value of the cell.

This command can be overridden for particular cells using the -changecmd option on mapcell and its peers.

-colorigin num
The numeric index of the first column in the sheet. The option defaults to 0, but it is common to redefine it so that the -titlecols have negative indices and the first data column has index 0. For example, if -titlecols is 1, then this would be set to -1.
-cols columns
The total number of columns of data in the table, including title columns.
-font font
The default font; cell options can override this for specific cells.
-formatcmd cmd

Specifies a command to format cell model values for display. The cmd must be a command prefix taking one additional argument, the value to format, and returning the formatted output.

This command can be overridden for particular cells using the -formatcmd option on mapcell and its peers.

-height rows
The number of visible rows in the table. If the value is 0, then the widget will attempt to display all -rows rows.
-maxheight pixels
The maximum height of the widget in pixels; the user must scroll to see any content that doesn't fit in this space. Defaults to the height of the screen.
-maxwidth pixels
The maximum width of the widget in pixels; the user must scroll to see any content that doesn't fit in this space. Defaults to the width of the screen.
-refreshcmd cmd
A Tcl script that is called whenever the widget has just refreshed its contents. The command is usually used to update Tktable tag configurations so as to change the appearance of particular cells. For example, it can change the color problem cells to red.
-roworigin num
The numeric index of the first row in the sheet. The option defaults to 0, but it is common to redefine it so that the -titlerows have negative indices and the first data column has index 0. For example, if -titlerows is 1, then this would be set to -1.
-rows rows
The total number of rows of data in the table, including title rows.
-solvecmd cmd
The cmsheet(n) calls command to recompute the -cellmodel whenever the user enters a valid input in a cell. By default, the cmsheet(n) will simply recompute the entire cell model; this command can be used to limit the computation to particular pages of the cell model.
-state state
If set to normal (the default) editable cells are editable; if set to disabled, they are not. Note that this has no effect on programmatic updates to the widget. Defaults to normal.
-titlecols num
The number of title columns. Title columns do not scroll, and are tagged with the title tag. Defaults to 0.
-titlerows num
The number of title rows. Title rows do not scroll, and are tagged with the title tag. Defaults to 0.
-validatecmd cmd

Specifies a command to call to validate user input. The command should be a prefix to which will be added two arguments, the Tktable "r,c" index of the edited cell, and the edited value. The command must either return the (possibly modified) value, or throw an error with error code INVALID. Note that if the table cell is mapped to a cellmodel(n) cell, you can retrieve the cell name using the cell method.

By default, cmsheet(n) requires all entered values to be valid real numbers.

This command can be overridden for particular cells using the -validatecmd option on mapcell and its peers.

-width columns
The number of visible columns in the table. If the value is 0, then the widget will attempt to display all -cols columns.
-xscrollcommand cmd
The usual Tk X-scrollbar command.
-yscrollcommand cmd
The usual Tk Y-scrollbar command.

COMMANDS

This module defines the following command:

cmsheet pathName ?options...?
Creates a new cmsheet(n) widget named pathName.

INSTANCE COMMAND

win cget option
Returns the value of the specified option.
win configure option value...
Sets the values of the specified options. Most changes will be displayed automatically; however, for some it might be necessary to refresh the widget.
win cell index
Given a Tktable cell index, returns the name of the cellmodel(n) cell that is mapped to it, or "" if none.
win empty rc0 rc1
Tells the cmsheet(n) that the rectangle of cells with upper-left corner specified by Tktable index rc0 and lower-right corner specified by Tktable index rc1 contain no data and should be tagged with the empty tag. By default, empty cells are flat and have the default GUI background color; however, this can be changed using the tag command.
win index index
Given a Tktable cell index or a cellmodel(n) cell name, returns the Tktable "r,c" index of the cell.
win mapcell rc cellname tag ?options...?

Maps the cellmodel(n) cell named cellname to the Tktable cell with index rc. The Tktable cell will display the value of the cell model cell. If the cell model cell is a constant, then by default the Tktable cell will be editable; otherwise not. The Tktable cell will be tagged with the named tag; if tag is %cell, then it will tagged with the cellname. In either case, the tag can then be configured with a variety of options. In addition to the Tktable tag options (see the Tktable documentation), cmsheet(n) provides the following:

-changecmd cmd
Specifies a command to be called whenever the cell is successfully edited. The cmd must be a command prefix taking two additional arguments, the name of the cell in the cellmodel(n) and the new value of the cell.
-formatcmd cmd
The cmd is a command prefix taking one argument, the cell value to be formatted. Whenever the mapped cell's value is to be displayed, this command will be used to format it for display.
-validatecmd cmd

Specifies a command to call to validate user input. The command should be a prefix to which will be added two arguments, the Tktable "r,c" index of the edited cell, and the edited value. The command must either return the (possibly modified) value, or throw an error with error code INVALID. Note that if the table cell is mapped to a cellmodel(n) cell, you can retrieve the cell name using the cell method.

By default, cmsheet(n) requires all entered values to be valid real numbers.

win map rc indx jndx pattern tag ?options...?

Maps a set of cell model cells to a rectangular region of the sheet. The set of cell model cells is defined by two cellmodel(5) indices, indx and jndx, and a pattern. The cells will be mapped to the table starting at index rc. The tag and options are defined as for mapcell.

Consider the following cell model: 

    index i {a b c}
    index j {a b c}

    forall i {
        forall j {
           let X.$i.$j = 0.0
        }
    }

This model defines a set of cells, X.a.a, X.a.b, and so on. The following command maps these nine cells to a 3x3 region of the sheet, with upper-left-corner at 1,1. The values will be editable (since the cells are constant cells, rather than formulas) and formatted with two decimal places: 

    $win map 1,1 i j X.%i.%j %cell \
        -formatcmd {format %.2f}

The i values will be mapped to the rows, and the j values to the columns.

Note that cellmodel(5) index names, i and j, appear as placeholders in the pattern, prefixed with "%". map determines the cell names by replacing the placeholders with each of the given index's values.

win mapcol rc indx pattern tag ?options...?
The mapcol method is similar to map, but maps cells down a single column. Thus, it has only one index, indx.
win maprow rc jndx pattern tag ?options...?
The maprow method is similar to map, but maps cells along a single row. Thus, it has only one index, jndx.
win see index
Scrolls the table so that the specified index is visible. The index map be a Tktable index or the name of a mapped cell model cell.
win refresh
Reloads, formats, and displays all mapped values from the underlying cell model.
win tag subcommand args...

This command is used to define, configure, and query Tktable tags; The mapcell and textcell methods, and their peers, use this command internally when mapping text and cells. It is sometimes useful to use it explicitly, e.g., to change cell colors in the -refreshcmd.

The command is delegated to the underlying Tktable widget; see the Tktable documentation for details.

win textcell rc text ?tag ?options...??
Places the text in the cell with Tktable index rc. (Note that it is not allowed to write text to a Tktable cell mapped to a cellmodel cell). If desired, the text can be styled with a tag, and that tag can be configured with any desired Tktable tag options. See the Tktable documentation for the available options.
win textcol rc textlist ?tag ?options...??
Like textcell, but writes successive elements from list textlist down the cells in a column starting at index rc.
win textrow rc textlist ?tag ?options...??
Like textcell, but writes successive elements from list textlist along the cells in a row starting at index rc.
win width args...
Use this command to set or query column widths. See the Tktable documentation for more information.
win window args...
Use this command to embed widgets in table cells. See the Tktable documentation for more information.
win xview args
win yview args
These are the usual Tk scrolling commands. See the Tktable documentation if need be, but they work just like the equivalent commands on the standard Tk widgets.
