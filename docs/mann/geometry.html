DESCRIPTION

geometry(n) is a package of computational geometry routines. The optional Marsbin binary extension includes fast C versions of the more computationally-intensive commands.

GEOMETRIC FIGURES

Points

A point is a x,y-coordinate pair of decimal numbers represented as a Tcl list with two elements. In practice, the X and Y coordinates are often longitude and latitude in decimal degrees; however, the code in this module makes no such assumption.

For example, the following creates the point corresponding to the coordinate pair (1.0, 2.0): 

    set p1 {1.0 2.0}

The point, px, and py commands are defined for use in creating and manipulating individual points.

Polygons and Polylines

A polygon is a closed sequence of line segments defined by a series of points; a polyline, often simply called a line, is an open sequence. Both polygons and polylines are represented as a flat Tcl list of coordinates. Thus, the following list could represent a three-sided polygon (a triangle) or a polyline consisting of two line segments: 

    set coords {0 0  1 0  1 1}

Note that for polygons, the points should be listed in counter-clockwise order. All polygons should be simple--i.e., adjacent edges should intersect only at one endpoint, and non-adjacent edges should not intersect at all--but they may be concave.

The cindex, clength and creverse commands are defined for manipulating lists of coordinates.

COMMANDS

geometry(n) defines the following commands, apart from those defined in the submodules listed above.

bbox coords

Returns the bounding box of a list of X,Y coordinates as a list of four elements: xMin yMin xMax yMax.

The binary extension includes a fast C implementation of bbox.

boxaround radius x y
Returns a box a x1 y1 x2 y2 with radius radius centered at x, y.
ccw a b c

This function is the "counterclockwise" primitive, as defined on page 350 of Sedgewick's Algorithms in C (Addison-Wesley, 1990) and corrected by Paul Firnette for use in the Corps Battle Simulation. Given three points a, b, and c, it determines whether a path from a to b to c turns counterclockwise or clockwise. Most of the complexity is there to handle degenerate cases: when the three points are collinear, or two or more points are identical.

Nominally, ccw returns 1 if the path turns counterclockwise, -1 if it turns clockwise, and 0 in certain degenerate cases. The degenerate cases are handled as follows:

  • If a == b, assume clockwise (-1) if c is to the left of a on the X-axis, counterclockwise (+1) if c is the right of a on the X-axis, and 0 otherwise. (Thus, ccw returns 0 if all three points are equal.)
  • If the three points are collinear in the order c, a, b, return clockwise (-1).
  • If the three points are collinear in the order a, b, c, return counterclockwise (+1). Yes, if the path does not in fact turn at all, we pretend it turns counterclockwise.
  • In all other cases, return 0.

The binary extension includes a fast C implementation of ccw.

cedge coords i

Retrieves the ith edge from a polygon represented as a flat list of X,Y coordinates. Edges are indexed from 0 to N-1; however, indices less than zero or greater than N-1 wrap around appropriately.

The edge is returned as a list of four coordinates, {x1 y1 x2 y2}.

cindex coords i

Retrieves the ith point from a flat list of X,Y coordinates. Points are indexed from 0 to N-1, where N is computed using clength. Note that if i is greater than or equal to N it wraps around, as this is often a useful property when dealing with polygons: 

    set coords {0 0   1 1   2 2}
    set a [cindex $coords 0]  ;# a = {0 0}
    set b [cindex $coords 1]  ;# b = {1 1}
    set c [cindex $coords 2]  ;# c = {2 2}
    set d [cindex $coords 3]  ;# d = {0 0}
clength coords
Returns the number of X,Y points in a list of coordinates, or half the [llength] of the list.
creverse coords
Returns the coordinates in reverse order, i.e., {1 2 3 4} becomes {3 4 1 2}. This can be used to put a clockwise polygon in counter-clockwise order.
intersect p1 p2 q1 q2

Determines whether the line segment from point p1 to p2 intersects the line segment from point q1 to q2. Handles all cases, including intersection at one or both endpoints and degenerate segments (p1 == p2, or q1 == q2, or both).

Returns 1 if the segments intersect, and 0 otherwise.

The binary extension includes a fast C implementation of intersect.

point x y
Returns a point, given coordinates x and y.
ptinpoly poly p ?bbox?

Determines whether or not point p falls inside or on the border of polygon poly, where poly is a flat list of X,Y coordinates. Returns 1 if so, and 0 if it is outside. If bbox is provided it should be the bounding box of the polygon, as computed by bbox; otherwise the bounding box is computed on the fly.

The binary extension includes a fast C implementation of ptinpoly.

px point
Returns the point's X-coordinate.
py point
Returns the point's Y-coordinate.

ENVIRONMENT

Requires Tcl 8.5 or later.

To use this package in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
