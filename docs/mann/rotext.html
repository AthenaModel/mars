DESCRIPTION

The rotext(n) widget provides read-only browsing and searching of styled text. As such, it is a drop-in replacement for the standard text(n) widget, with these differences:

  • The insert and delete methods are no-ops; use the new ins and del methods instead. These methods are used precisely like the text(n) methods they replace.
  • The navigation key-bindings have been changed to allow easier browsing, i.e., the Up and Down arrow keys scroll the text in the widget rather than moving the insertion point (which is hidden in any event).
  • The <<SelectAll>> virtual event will select all of the text in the widget.
  • The find method and -foundcmd callback work with the finder(n) widget to allow searching of the displayed text.

This man page describes only those options and subcommands which are specific to rotext(n); but see also the Tk text(n) man page for more details.

Text Tags

The rotext(n) widget defines the following text tags, which are used by the find mechanism:

  • FOUND: This tag is used to highlight all instances of a search target in the widget's text. By default, it adds boldface and underlining.
  • SHOWN: This tag is used to a line containing an instance of the search target. The find subcommands can be used to move the highlight from one instance to the next, scrolling the widget as needed. By default, it sets the foreground color to white and the background color to black.

COMMANDS

This module defines the following command:

rotext pathName ?options...?

Creates a new rotext widget named pathName.

The rotext widget may be created with the following rotext(n)-specific options, along with the normal text(n) options:

-foundcmd cmd

This option is used to link a rotext(n) widget with a finder(n) widget, or any other widget which supports the "found" half of the find/found protocol. It specifies a command which will be called when the widget's find status is updated. It will be called with two additional arguments, both integers: the number of instances of the target string which have been found (or 0 if there are none) and the index (0 to N-1) of the instance which is currently highlighted (or -1 if no instance is highlighted).

See finder(n) for an example.

INSTANCE COMMAND

$rotext configure option value...
Sets the values of the specified options.
$rotext cget option
Returns the value of the specified option.
$rotext del args...
Replacement for the text(n) "delete" method.
$rotext ins args...
Replacement for the text(n) "insert" method.
$rotext selectAll
Selects all of the text in the widget.
$rotext find subcommand ?args...?

The find method, along with the -foundcmd callback, implements the rotext(n) widget's half of the find/found protocol. It is used to search for and highlight text strings.

$rotext find target targetType target

Searches for all instances of target, which must be a string of the given targetType:

  • exact -- any text string
  • wildcard -- a glob-pattern
  • regexp -- a regular expression

All instances are highlighted with the FOUND tag. Then, the widget scrolls to show the line containing the last instance of the target; the line is highlighted with the SHOWN tag. Finally, the -foundcmd, if any, is called.

To clear the search, call this method with a target of "".

$rotext find update ?force?

This method causes the widget to redo the current search. If a line was highlighted before find update it will still be highlighted after unless force is set true.

Call this method after changing the contents of the widget.

$rotext find count

Returns the number of instances of the target which were found. Note that if the target occurs twice on one line, that counts as a single instance.

Returns 0 if there's no target.

$rotext find instance

Returns the index (0 to N-1) of the instance which is currently highlighted, or -1 if no instance is highlighted.

Note that the resulting value might be incorrect if the widget's contents have been modified since the search was done (unless find update has been called).

$rotext find show instance
Scroll to and highlight the line containing an instance of the search target. instance is the instance number, 0 to [find count]-1, or "end" for the final instance.
$rotext find prev
Scroll to and highlight the line containing the next instance of the search target, if any.
$rotext find next
Scroll to and highlight the line containing the previous instance of the search target, if any.

ENVIRONMENT

marsgui(n) requires Tcl 8.5 or later.

To use marsgui(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
