DESCRIPTION

The hbarchart(n) widget plots from one to ten series of data values as horizontal bars against a set of entities on the Y-axis. Individual series of data can be updated individually.

OPTIONS

The widget has the following options.

-height pixels
Specifies the height of the chart window.
-title text
Specifies an overall title for the chart. If text is the empty string (the default), no title is drawn.
-titlepos anchor
Specifies where to put the title; anchor may be n or s to center the title at the top or bottom. Defaults to n.
-width pixels
Specifies the widget of the chart window.
-xformat format
Specifies an format(n) conversion for formatting data values. Defaults to "%.1f".
-xtext text
Specifies a label for the X-axis. If text is the empty string (the default), no label is drawn.
-ylabels labels

The widget plots one or more data values for each of a list of entities, each one of which is shown on the Y-axis. The value of this option determines both the labels shown on the Y-axis and the number of entities.

If the value of this option is changed, the plotted data series are deleted, and will need to be re-plotted using the plot method.

-yscrollcommand command
If the widget displays too many data bars to be seen at once, its data area can be scrolled vertically in the usual way. Thus, the command would usually be to a vertical scrollbar's set method.
-ytext text
Specifies a label for the Y-axis. If text is the empty string (the default), no label is drawn.

COMMANDS

This module defines the following command:

hbarchart pathName ?options...?
Creates a new hbarchart widget named pathName.

INSTANCE COMMAND

win cget option
Returns the value of the specified option.
win configure option value...
Sets the values of the specified options. The widget will redraw the chart accordingly.
win plot name ?options...?

Plots a series of data values against each of the entities in -ylabels. The name is used to identify the series for future updates.

The options are as follows:

-data values
Specifies a list of numeric data values, one for each label in -ylabels.
-label text
Specifies a label for this series of data. The label is used in the legend. If no -label is given, the series name is used instead.
-rmin min
-rmax max
These options specify the range bounds for the series--that is, not the min and max values in the data, but the logical min and max value for the data type. Strict percentages, for example, have a range from 0.0 to 100.0. If one of these options is omitted or explicitly set to the empty string, the relevant bound is computed from the data.
win yview args...
The widget supports the standard canvas(n) yview method, to allow vertical scrolling.

VIRTUAL EVENTS

The hbarchart(n) widget generates the following virtual events.

<<Context>>

This event is generated when the user right-clicks or control-clicks on the chart. It supports the following substitutions: %x, %y, %X, %Y, and %d.

The value of the %d substitution is a list whose first element indicates what was clicked on; the remaining items in the list vary depending on the first element. The current possibilities are as follows:

none
The user right-clicked on nothing in particular.
bar ylabel series value
The user right-clicked on a data bar. The data identifies the bar by its ylabel and series name, and also gives the data value.

ENVIRONMENT

This package requires Tcl/Tk 8.5 or later.

To use this package in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
