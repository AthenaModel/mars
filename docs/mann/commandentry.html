DESCRIPTION

The commandentry(n) widget is a Tk entry widget customized to support a bash/tcsh-style command history, which is accessible by pressing the Up and Down arrow keys.

COMMANDS

This module defines the following command:

commandentry pathName ?options...?

Creates a new commandentry(n) widget named pathName.

The commandentry(n) widget may be created with the standard Tk entry widget options, plus the following commandentry-specific options:

-history num
Specifies the number of commands saved in the history. Defaults to 20.
-clearbtn flag

If true, the commandentry will have a "clear" button at its righthand end. The button looks like a black circle with an "X" in it; it will be enabled when the widget contains any text, and disabled otherwise. Defaults to false.

This option can only be set at creation time.

-returncmd cmd
Specifies a command that will be called when the text in the widget is executed, i.e., when the user presses <Return> or the application calls execute. A single argument, the current content of the widget, will be appended to the command.
-changecmd cmd
Specifies a command to be called when the contents of the widget changes due to user interaction. A single argument, the new content of the widget, will be appended to the command. Note that the set and clear methods never cause the -changecmd to be called.
-keycmd cmd

Specifies a command that will be called on every <KeyRelease>; event. The character code and the keysym will be appended to the command as two separate arguments.

Note that the test [string is print $char] is useful for determining whether the user entered a real character or some other key.

INSTANCE COMMAND

$commandentry set text
Sets the contents to text. The -changecmd will not be called.
$commandentry clear
Clears the commandentry. The -changecmd will not be called.
$commandentry get
Returns the current contents.
$commandentry execute
Saves the current contents of the widget to the history stack, unless the widget is empty or the current contents matches the previous entry in the history stack; then calls the -returncmd with the current contents.

ENVIRONMENT

marsgui(n) requires Tcl 8.5 or later.

To use marsgui(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
