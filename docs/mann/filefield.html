DESCRIPTION

The filefield(n) widget is a field(i) data entry field used for selecting files from the disk using the standard Tk tk_getOpenFile dialog. It provides a dispfield(n) to display the name of the selected file, and a "Browse" button that pops up the file selection dialog.

The display shows the base name of the selected file; but the absolute path of the file is shown in a tool tip. The value of the field is the absolute path.

COMMANDS

This module defines the following commands.

filefield pathName ?options...?

Creates a new filefield widget named pathName, and returns pathName. The widget supports the following options:

-changecmd command
From field(i). Specifies a command to be called whenever the field's value changes, for any reason whatsoever (including explicit calls to set. The new value is appended to the command as an argument.
-state state
From field(i). filefield(i) widgets have states, just as the normal tk:: and ttk:: widgets do. The valid states are normal and disabled.
-width chars
Specifies the width in characters of the embedded display field.
-title string
Specifies the title string for the file selection dialog. Defaults to "Select File".
-filetypes list
Specifies the acceptable file type or types, in the same form as for tk_getOpenFile. Defaults to any kind of file.

INSTANCE COMMAND

filefield(n) widgets have the following subcommands:

pathName cget option
From field(i). Returns the value of the specified option.
pathName configure option value...
From field(i). Sets the value of one or more of the object's options.
pathName get
From field(i). Returns the field's current value.
pathName set value
From field(i). Explicitly sets the field's value, and calls the -changecmd if the value has really changed, i.e., successive calls to set with the same value will result in at most one call to the -changecmd.
