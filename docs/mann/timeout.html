DESCRIPTION

timeout(n) defines an object that manages scheduling, repetition, and cancelling of "after" timeouts and idle handlers. The caller specifies the timeout interval, a Tcl script to call when the timeout fires, and whether the timeout should be rescheduled automatically. A scheduled timeout is cancelled via the cancel method or by destroying the object; note that the timeout callback can cancel a repeating timeout.

Note that the timeout is not automatically scheduled when the object is created; instead, the schedule method must be used explicitly.

COMMANDS

timeout name ?options?

Creates a new timeout(n) object called name. The object is represented as a new Tcl command in the caller's scope; timeout returns the fully-qualified form of the name.

The timeout may be created with the following options:

-command script
Defines a Tcl script to be evaluated in the global scope when the timeout fires. Defaults to "".
-interval idle|msecs

Specifies the timeout interval, which must be either the string idle or the length of the timeout interval in milliseconds (a positive integer). If set to idle, the object schedules idle tasks rather than timeouts. Defaults to 1000 (1 second).

If this value is changed after the object is created, then it takes effect the next time schedule is called.

-repetition boolean
If true, the timeout will be rescheduled automatically after each call of the -command, unless the -command calls the cancel method. Defaults to false.

INSTANCE COMMAND

Each instance of the timeout object has the following subcommands:

$timeout cget option
Returns the value of the specified option.
$timeout configure option value...
Sets the value of one or more of the object's options.
$timeout schedule ?-nocomplain?
Schedules the timeout to fire after -interval milliseconds from the present time. Normally it's an error to call schedule if the timeout already isScheduled; if -nocomplain is specified, then schedule is a no-op if the timeout isScheduled.
$timeout cancel
Cancels any scheduled timeout; if called by the -command, prevents -repetition. After cancel is called, the timeout can only be scheduled for execution by calling schedule. Note that it is never an error to call cancel.
$timeout isScheduled
Returns 1 if the timeout is scheduled, and 0 if not.

ENVIRONMENT

Requires Tcl 8.5 or later.

To use this package in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
