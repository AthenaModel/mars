DESCRIPTION

The loglist(n) widget is used to browse the applications and application log files in a log directory tree. The directory tree has this structure:

rootdir/appdir/logNNNNN.log

where rootdir is the root of the tree. The rootdir may contain zero or more application subdirectories; each application may have written zero or more log files.

The loglist(n) widget displays two lists in a vertical column: a list of application names and a list of log files for the currently selected application. Clicking on an application name in the upper list displays that application's log files in the lower list. Clicking on a log file causes the -selectcmd to be called for the selected log file.

loglist(n) also supports searching across an application's log files; see searchlogs for more details.

COMMANDS

This module defines the following command:

loglist pathName ?options...?

Creates a new loglist widget named pathName.

The loglist widget may be created with the following options:

-width chars
The width of the loglist in character cells. Defaults to 25. Read-only after creation.
-height chars
The approximate height of the loglist in character cells. The application list and the log file list will each be allocated roughly half of the -height. Defaults to 15. Read-only after creation.
-showtitle boolean
When true (the default), a button containing -title will be included directly above the list. Pressing the button causes the list to be updated.
-showapplist boolean
When true (the default), the list of applications will be included directly above the list of logs files.
-rootdir directory
The root of the log tree; its subdirectories contain log files for specific applications.
-logpattern glob_pattern
The glob pattern used to filter files in the application directories. Files matching the pattern are assumed to be log files; all other files are ignored. Defaults to "*.log".
-msgcmd cmd
The command used for reporting progress and status; it will usually write to the window's message line. The message will be appended to the cmd as a single string argument. Defaults to "".
-selectcmd cmd
A command to execute whenever a log file is selected. Two arguments will be appended: The file's full pathname followed by a boolean flag indicating if the file changed or not. When there is no currently-selected log file, the filename will be passed the empty string, "". Note that a true value for the changed flag might be set true even if there was no change. This should be treated as a hint that the user is requesting an update.
-title title
Specifies the title which appears on the update button at the top of the widget. Pressing the button updates the contents of the widget. Defaults to "Applications:".
-autoload boolean
If enabled, specifies that the most recent file should be loaded during an update. An update in this context refers to either a manual or automatic update. The default is true.
-autoupdate boolean
If enabled, the loglist(n) updates its contents every -updateinterval. Defaults to "off". Note that this option can be enabled and disabled at any time. See update for details on what happens during an update.
-updateinterval msecs
Sets the auto-update interval to some number of milliseconds. Defaults to 1000 (one second).
-defaultappdir appdir
Sets the default application name; that application's subdirectory will be selected automatically when the loglist is first updated.
-formattext val
A boolean option that indicates whether or not log entries should be formatted during a call to searchlogs. This only makes sense if -parsecmd and -formatcmd are also defined. Searching with this option active will generally be much slower than without it.
-parsecmd cmd
A parsing command used to parse the contents of a log file into the proper format. The parsecmd will be passed a single argument, the text string to be parsed; this string will represent a block of zero or more log entries. This option only matters when -formattext is true.
-formatcmd cmd
A formatting command to convert a list of parsed log entries into a list of formatted log entries just as they would be displayed currently. It's up to client to define what formatted means. This option only matters when -formattext is true.
-filtercmd cmd
A filtering command to filter individual entry strings. The command should take the entry string as the single argument. The command should return 1 if the entry string passes the filter and 0 otherwise. This command will be used during calls to searchlogs when -formattext is off.

INSTANCE COMMAND

$loglist configure option value...
Sets the values of the specified options.
$loglist cget option
Returns the value of the specified option.
$loglist update

On update, the loglist(n) looks for new log files for the currently selected application, if any, and automatically selects the most recent one; the -selectcmd is called in either case. In addition, the loglist(n) looks for and displays any new application directories.

If no log files are found, whether because the current application directory is empty or because it has ceased to exist, the -selectcmd will be called with the empty string, "".

$loglist searchlogs direction target ?searchtype?

Executes a search for the target string among the current application's log files. The kind of search is determined by the searchtype, which may be exact (the default), wildcard, or regexp. Note that searches are done using grep(1), so fancy regular expressions may not work quite the same as they do for searches done using Tcl's regexp matcher.

The search takes the current log file as the starting point and steps through the earlier or later logs as specified by the direction.

The first log in the specified direction to contain a hit is automatically selected, and the -selectcmd is called. If no hits are found in that direction then a message is written via the -msgcmd.

Returns 1 on success and 0 on failure.

Searching can be a lengthy process; calling stopsearch will terminate it prior to completion.

$loglist stopsearch
Stops the active log search, if there is one.

ENVIRONMENT

marsgui(n) requires Tcl 8.5 or later.

To use marsgui(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
