SYNOPSIS

package require marsgui 3.0.23
namespace import ::marsgui::*
  • winbrowser pathName ?options...?
  • $winbrowser configure option value...
  • $winbrowser cget option
  • $winbrowser refresh

DESCRIPTION

marsgui(n) contains general-purpose (i.e., non-application-specific) GUI code for use in Mars. This module defines the ::marsgui::winbrowser widget, a development tool for browsing the Tk window hierarchy. The widget is used by the debugger(n) component.

The winbrowser(n) component has two resizeable panes, a tree of Tk windows, starting with ".", and a data pane which displays information about the currently selected window. The data pane contains a tabbed notebook with three tabs: winfo, options, and bindings. The winfo tab contains general information about the window and its geometry, mostly derived from the Tk winfo command. The options tab contains all of the window's (known) options and values. The bindings tab lists the window's bind tags, and then the bindings and their scripts for each bind tags.

OPTIONS

The widget has the same options as the standard ttk::frame; in addition, it provides the following:

-logcmd command
Allows the widget to display isearch(n) status messages to the user. The command is a Tcl command expecting one additional argument, a text string. The client will usually arrange for this string to be written to a messageline(n).

COMMANDS

This module defines the following command:

winbrowser pathName ?options...?
Creates a new winbrowser widget named pathName.

INSTANCE COMMAND

$winbrowser configure option value...
Sets the values of the specified options.
$winbrowser cget option
Returns the value of the specified option.
$winbrowser refresh
Refreshes the data displayed by the winbrowser(n) widget.

ENVIRONMENT

marsgui(n) requires Tcl 8.5 or later.

To use marsgui(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
