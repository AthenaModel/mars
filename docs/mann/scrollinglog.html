DESCRIPTION

This module defines the ::marsgui::scrollinglog widget, which is used to browse periodically updated text files in the manner of tail -f. By default, the widget is configured to browse logger(n) debugging logs; however, the columns, file parsing, and highlight colors can be configured to support many different formats of text file.

scrollinglog(n) combines the display capabilities of the logdisplay(n) widget with the search and filtering capabilities of the finder(n) and filter(n) widgets in one convenient package. A loglist(n) widget can be optionally included.

A combobox is provided to filter based on log level. The select log level and higher levels will be displayed. The levels provided are in accordance with those supported by logger(n).

An auto-scroll/scroll-lock selection button is provided in the upper right-hand corner. In the scroll-lock position, the contents will remain fixed and under the user's control. In addition, an enabled scroll-lock will prohibit load from loading the file contents.

COMMANDS

This module defines the following command:

scrollinglog pathName ?options...?

Creates a new scrollinglog widget named pathName.

The scrollinglog widget may be created with the following options:

-showloglist boolean
When true, enables the inclusion of a loglist(n) widget to the left of the logdisplau(n) widget. When enabled the loglist(n) will be configured according to these options: -defaultappdir, -formattext, -logpattern, -rootdir, -parsecmd and -updateinterval.
-logcmd command
Defines a command the scrollinglog can call if it needs to output a message. One argument will be appended to the command, the message to be output.
-loglevel level
Defines the maximum log verbosity level to display. The value of level must be one of fatal, warning, normal, detail or debug. See logger(n) of marsutil for more information on log levels.
-title title
Obsolete. This option no longer has any effect.
-updateinterval milliseconds
Determines how often the scrollinglog will perform automatic updates when auto-scroll is enabled. Defaults to 1000 (i.e., one second).
-autowidth boolean
-format formatlist
-parsecmd cmd
-tags tagslist
-showtitle boolean
These options are all identical to the matching logdisplay(n) options; they are configured by default to support logger(n) log files. See the logdisplay(n) man page and the scrollinglog(n) code to see how to customize them for other formats. If -showloglist is true, the -parsecmd and -showtitle options will also be used to configure the loglist according to loglist(n).
-font font
-height height
-width width
-foreground color
-background color
-insertbackground color
-insertwidth color
These options are all identical to the Tk text widget options of the same names.
-borderwidth width
-relief relief
-defaultappdir appdir
-formattext val
-logpattern glob_pattern
-rootdir directory
-showapplist boolean
These options are all identical to the matching loglist(n) options; they are configured by default to support logger(n) log files. See the loglist(n) man page and the scrollinglog(n) code to see how to customize them for other formats.

INSTANCE COMMAND

$scrollinglog configure option value...
Sets the values of the specified options.
$scrollinglog cget option
Returns the value of the specified option.
$scrollinglog load filename
Tells scrollinglog to display the named file unless scroll-lock is currently enabled. In this case, the file name is cached for later use.

ENVIRONMENT

marsgui(n) requires Tcl 8.5 or later.

To use marsgui(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
