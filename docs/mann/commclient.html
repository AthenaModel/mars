DESCRIPTION

commclient(n) defines a client for connecting to a commserver(n) object in another application over a TCP/IP socket.

The client can send commands to the server for processing; it can also receive update commands from the server. Update commands are processed in a safe interpreter; it is the client's responsibility to alias handlers for each update command into the safe interpreter using the alias method.

The connection is first established using the connect command; if the first attempt is unsuccessful, commclient(n) will retry every -retrymsecs milliseconds until it is successful or the object is destroyed. Similarly, once connected commclient(n) will ping the commserver(n) every -pingmsecs milliseconds, to verify that it is still there. Should the commserver(n) goes away, commclient(n) will begin to try to re-establish the connection as soon as the server's disappearance is noted.

The -connectcmd, if any, is called when the client first succeeds in connecting to the server; the -disconnectcmd is called if the server disconnects, as soon as the server's disappearance is noted.

COMMANDS

commclient name ?options?

Creates a new commclient(n) object called name. The object is represented as a new Tcl command in the caller's scope; commclient returns the fully-qualified form of the name.

The commclient may be created with the following options:

-clientname name
Logical name of the client. Defaults to ""; it needs to be set only if the server implements logical-name-based validation. The set of valid logical names is then determined by the server.
-connectcmd command
When commclient(n) first succeeds in connecting to a commserver(n), and on successful reconnection, commclient(n) will call the specified command with no additional arguments.
-disconnectcmd command
When commclient(n) loses its connection to the commserver(n), it will call the specified command with no additional arguments. Note that command is not called on unsuccessful connection attempts.
-logcomponent name
The component name to pass to the -logger object when logging messages; defaults to "commclient".
-logger name

The name of the logger(n) object to use to log client activity.

This option must be set at creation time, and is read-only thereafter.

-retrymsecs interval
Specifies the connection retry interval in milliseconds; defaults to 500. Must be positive.
-pingmsecs interval
Specifies the ping interval in milliseconds; defaults to 5000. Must be positive.
-portid portId

The comm(n) port ID of the commserver(n) to which the client will connect.

This option must be set at creation time, and is read-only thereafter.

-hostip hostIp

The IP address of the jnem_sim(1) host, or "" for the local host.

This option must be set at creation time, and is read-only thereafter.

INSTANCE COMMAND

Each instance of the commclient object has the following subcommands:

$commclient cget option
Returns the value of the specified option.
$commclient configure option value...
Sets the value of one or more of the object's options.
$commclient connect

Attempts to connect to the server; this generally needs to be called only once, as soon as the client is ready to connect. If the server does not respond, the client will retry every -retrymsecs milliseconds. An error is thrown if the server refuses the connection.

It's an error to call connect if the client is already connected to the server.

$commclient send script

Sends the script to the server for execution. If the server executes the script normally, send will return the script's result; if the server catches an error while executing the script, then send will throw the same error message (the -errorCode is not preserved).

If send is called when the client is not yet connected to the server, this error will be thrown: "client is not connected to the server".

If send tries to send the script and fails because the server has disconnected, this error will be thrown: "lost connection".

$commclient id
Returns the client's comm(n) ID.
$commclient alias srcCmd targetCmd ?arg...?
Aliases the target update command into the safe interpreter as srcCmd.

To use this package in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
