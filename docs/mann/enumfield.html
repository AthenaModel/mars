DESCRIPTION

The enumfield(n) widget is a ttk_combobox(n) configured as a field(i) data entry field. It is used for choosing items from a (possibly dynamic) list of arbitrary text strings.

COMMANDS

This module defines the following commands.

enumfield pathName ?options...?

Creates a new enumfield widget named pathName, and returns pathName. The widget supports the following options:

-autowidth flag
If true, the widget will compute its own width based on the -values, however specified. If false (the default) the width will not be based on the width of the content.
-background color
Sets the color of the underlying hull frame, which is invisible unless the -borderwidth is non-zero.
-borderwidth width
Sets the width of the widget border in pixels (or other screen dimensions). Defaults to 0.
-changecmd command
From field(i). Specifies a command to be called whenever the field's value changes, for any reason whatsoever (including explicit calls to set. The new value is appended to the command as an argument.
-state state
From field(i). enumfield(i) widgets have states, just as the normal tk:: and ttk:: widgets do. The valid states are normal and disabled.
-displaylong flag
If -displaylong is set, then the widget manages two sets of values: the application values, used by get and set, and the displayed values, that appear in the field and the pulldown. Where the values come from depends on the -enumtype, -valuecmd, and -values options.
-enumtype enumType

Specifies the name of an enum(n) type, or some other object that presents a similar interface. The widget will dynamically retrieve its list of valid values by calling 

    {*}$enumType names

If -displaylong is true, then the widget will retrieve the displayed values by calling 

    {*}$enumType longnames

If -enumtype is specified, then -valuecmd and -values are ignored.

-valuecmd command

Specifies the name of a command to call to dynamically retrieve the list of valid values. The -enumtype option takes precedence.

-displaylong is (currently) ignored if this option is used. TBD: this should be fixed, if -valuecmd is ever actually used in the application.

-values values

Statically defines the list of valid values. If either -enumtype or -valuecmd are specified, this option is ignored.

If -displaylong is true, then values is a flat list of application value/displayed value pairs.

-width width
Specifies the approximate width of the widget in characters of the current font.

INSTANCE COMMAND

enumfield(n) widgets have the following subcommands:

pathName cget option
From field(i). Returns the value of the specified option.
pathName configure option value...
From field(i). Sets the value of one or more of the object's options.
pathName get
From field(i). Returns the field's current value.
pathName set value ?-silent?

From field(i). Explicitly sets the field's value.

By default, this command calls the client's -changecmd if the value has really changed, i.e., successive calls to set with the same value will result in at most one call to the -changecmd.

If the -silent option is given, however, the -changecmd is never called. (Note that -silent is not included in the field(i) interface.)

ENVIRONMENT

marsgui(n) requires Tcl 8.5 or later.

To use marsgui(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
