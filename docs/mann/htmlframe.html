DESCRIPTION

The htmlframe(n) widget is an htmlviewer(n) widget configured for use as a layout manager. The client creates any number of widgets as children of the htmlframe(n), and then specifies the layout as an HTML string.

The child widgets appear in the HTML string as <input> elements whose name attribute is the last component of the child widget's name. If the widgets are created like this, for example, 

    htmlframe $hf
    ttk::label $hf.hello -text "Hello, world!"
    ...

then the "hello" widget is layed out like this: 

    <input name="hello">

The size of the widget can be set explicitly in the usual way, when the widget is created; alternatively, the widget's width in characters can be set using the <input>'s size attribute.

The client may also wish to set the id, class, and style attributes.

Dynamic Layout

The layout can be changed at any time, simply by passing a new HTML string to the layout method. Display of particular elements can be changed dynamically by using the set method to set the class or style of particular elements. Elements are identified by an ID string defined as an element attribute. The widget's set and get method look for an id as follows:

  • A <label> element with for="id"
  • Any element with id="id"
  • Any element with name="id"

In this regard, it is often useful to define new CSS classes using the -styles option. For example, suppose certain element contains an error, and so should be rendered with red text; or suppose that a certain element should sometimes be hidden. These can easily be handled, as follows.

First, add the relevant CSS classes: 

    $hf configure -styles {
        .hidden { display: none; }
        .error  { color: red; }
    }

Second, write an HTML layout that includes id attributes for the elements you want to affect dynamically. 

    <span id="element1">
    I can say hello:
    <input name="hello">
    </span>

Then, setting the class of the element1 element will make the text change color or the whole element disappear: 

    $hf set element1 class hidden     ;# Hide it
    $hf set element1 class error      ;# Turn it red
    $hf set element1 class ""         ;# Set it back to normal

COMMANDS

This module defines the following commands.

htmlframe pathName ?options...?

Creates a new htmlframe widget named pathName, and returns pathName. The widget supports all of the htmlviewer(n) options; however, the ones most likely to be of interest are the following:

-background color
Sets the background color of the widget; the background defaults to the default GUI background color.
-hyperlinkcmd command
Specifies a command to be called if the user clicks on a hyperlink. The command will be called with one additional argument, the link's URL. It is up to the client to handle the link in some way.
-shrink flag
The widget is configured with -shrink set to yes by default; it will be just big enough to hold its contents.
-styles css
Specifies a CSS style sheet to add to the widget's default styles. This style sheet overrides the built-in defaults, and can be overridden by <style>...</style> sheets and style attributes in the input.

INSTANCE COMMAND

The htmlframe(n) widget has all of the subcommands of the htmlviewer(n) widget, and adds the following:

pathName cget option
Returns the value of the specified option.
pathName configure option value...
Sets the value of one or more of the object's options.
pathName layout html
Lays out the children of the widget using the HTML layout. Any <input>'s without matching child widgets are ignored.
pathName set id attribute value

Sets the value of the named attribute for the first HTML element found with the given id, where the element is found as follows:

  • Any element with id="id"
  • Any element with name="id"

If no such element is found, no change is made. This is convenience wrapper around htmlviewer(n)'s search and attribute subcommands.

pathName get id attribute

Gets the value of the named attribute for the HTML element with the given id, where the element is found as for the set command. If no element is found, the command returns the empty string.

This is convenience wrapper around htmlviewer(n)'s search and attribute subcommands.

ENVIRONMENT

marsgui(n) requires Tcl 8.5 or later.

To use marsgui(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
