DESCRIPTION

The finder(n) widget is used for controlling searching within other widgets, including the rotext(n), logdisplay(n) and loglist(n) widgets. It allows the user to enter a target string; it provides a menu for selecting the target type, which may be incremental, exact, wildcard, or regexp; and it provides buttons for navigating through the instances of the target in the data being searched.

See the EXAMPLE, below, for details on how to connect a finder(n) with a rotext(n) widget.

COMMANDS

This module defines the following command:

finder pathName ?options...?

Creates a new finder widget named pathName.

The finder widget may be created with all the standard Tk frame widget options, plus the following finder(n)-specific options:

-findcmd cmd
Specifies the command the finder will use to request searches and browse the search results. This will usually be the "find" method of a rotext(n) or similar widget. The EXAMPLE shows how to connect a finder(n) with a rotext(n) widget.
-width chars
Specifies the character width of the finder(n)'s text entry field. Defaults to 20.
-msgcmd cmd
Specifies a command to be used for passing messages to the user. The command will be called with one additional argument, the message to display; it will usually be displayed in the window's message line.
-targettype type
Sets the type of the target string: incremental, exact, wildcard, or regexp. Defaults to exact. If the target type is incremental, then an exact-match search will be done incrementally as each character of the target string is entered by the user.
-loglist widget
If set to the name of a loglist(n) widget, the finder(n) will include buttons for searching through the logs displayed in the loglist(n) using that widget's searchlogs and stopsearch methods.
-multicol val
A boolean option indicating whether or not the search string spans multiple columns. The intdended receiver of this will be the declared -loglist if it exists. If -loglist is not specified, this option has no use.

INSTANCE COMMAND

$finder found count instance

This method is used by the recipient of the -findcmd to notify the finder(n) of the search results.

The count is the number of instances of the target which were found; it will be 0 if no instances were found, or if the target is "".

The instance identifies the instance which is currently being highlighted on the display, if any. If an instance is highlighted, it will be a number from 0 to count - 1. Otherwise it will be -1.

The EXAMPLE shows how to connect the finder(n) to other widgets.

EXAMPLE

The finder(n) can be connected to any widget that supports the "find/found" protocol, e.g., the rotext(n) and logdisplay(n) widgets. Here's a short script that connects a finder(n) to a rotext(n) displaying an arbitrary text file. 

    #!/bin/sh
    # -*-tcl-*-
    # The next line restarts using wish \
    exec wish8.5 "$0" "$@"

    package require gui
    namespace import ::marsutil::* ::marsgui::*

    # FIRST, create a toolbar and a finder, and pack them on the top
    frame .bar

    finder .bar.finder               \
        -width   15                  \
        -findcmd [list .rotext find] \
        -msgcmd  puts

    pack .bar.finder -side right
    pack .bar -side top -fill x -expand 1

    # NEXT, create a rotext and fill in the rest with it.
    rotext .rotext                                   \
        -setgrid            1                        \
        -wrap               none                     \
        -highlightthickness 1                        \
        -borderwidth        1                        \
        -relief             sunken                   \
        -yscrollcommand     [list .yscroll set]      \
        -foundcmd           [list .bar.finder found]

    ttk::scrollbar .yscroll           \
        -orient vertical              \
        -command [list .rotext yview]

    pack .yscroll -side right -fill y
    pack .rotext -side right -fill both -expand 1

    # NEXT, load some text into the rotext
    if {[llength $argv] == 0} {
        puts "Usage: viewer filename"
        exit
    }

    set f [open [lindex $argv 0]]
    .rotext ins end [read $f]
    close $f

    .rotext see 1.0

ENVIRONMENT

marsgui(n) requires Tcl 8.5 or later.

To use marsgui(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
