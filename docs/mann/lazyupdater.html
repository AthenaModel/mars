lazyupdater(n) -- Lazy update manager, submodule of marsutil(n)

SYNOPSIS

package require marsutil 3.0.23
namespace import ::marsutil::lazyupdater
  • lazyupdater name ?options?
  •     -command script
  •     -delay idle|msecs
  •     -window window
  • obj cancel
  • obj cget option
  • obj configure option value...
  • obj update

DESCRIPTION

lazyupdater(n) defines an object, based on timeout(n), that allows an object to update itself in a lazy fashion. The client calls the update method whenever an update is desired; after a specified -delay, the -command is called. The update method can be called any number of times during the delay interval, but the -command will be called only once.

If the controlled object is a Tk widget, use the -window option; the -command will be called only when the window is mapped, thus saving unnecessary updates.

COMMANDS

lazyupdater name ?options?

Creates a new lazyupdater(n) object called name. The object is represented as a new Tcl command in the caller's scope; lazyupdater returns the fully-qualified form of the name.

The lazyupdater may be created with the following options:

-command script
Defines a Tcl script to be evaluated in the global scope when the lazyupdater fires. Defaults to "".
-delay idle|msecs

Specifies the update delay, which must be either the string idle or the length of the delay in milliseconds (a positive integer). If set to idle, the object schedules an idle task rather than a timeout. Defaults to 1.

If this value is changed after the object is created, then it takes effect the next time update is called.

-window window
If the -window option is set, it must be the name of a Tk widget. Then, the -command will be called only if the widget is mapped (i.e., visible), and an update will be scheduled automatically when the widget becomes mapped.

INSTANCE COMMAND

Each instance of the lazyupdater object has the following subcommands:

obj cancel
Cancels any pending update. This is useful when the client has forced an update for some reason.
obj cget option
Returns the value of the specified option.
obj configure option value...
Sets the value of one or more of the object's options.
obj update
Schedules the -command to be called after the -delay. This method can be called any number of times during the delay interval; the -command will be called only once.

ENVIRONMENT

Requires Tcl 8.5 or later.

To use this package in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
