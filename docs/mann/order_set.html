DESCRIPTION

The ::marsutil::order_set class is part of the Mars Order Processing framework, which includes order(n), order_set(n), order_flunky(n), and order_dialog(n). See order(n) for an overview of the framework.

An order_set(n) instance is used to define a collection of related order classes, which will all be subclasses of order(n) or an equivalent. As part of defining the order classes, order_set(n) ensures that each class defines the required metadata and does some other load-time checks.

COMMANDS

The following commands create instances of order_set(n):

::marsutil::order_set create name ?baseClass_? ?autoVars_?

Creates a new instance of ::marsutil::order_set called name. By default, orders defined using this class will have ::marsutil::order as their superclass; if baseClass_ is given, it will be used instead.

It is common for to baseClass_ to define an instance variable or two that will be used by most or all orders. Normally, each order class would have to declare this variable explicitly, which is tiresome. If autoVars_ is given, it should be a list of instance variable names; each name will be automatically declared as a variable for each defined order class, thus bringing the variable into scope in all of the order class's methods.

::marsutil::order_set new ?baseClass?
Creates a new instance of ::marsutil::order with an arbitrary name. By default, orders defined using this class will have ::marsutil::order as their superclass; if baseClass is given, it will be used instead.

INSTANCE COMMAND

Each instance of order_set(n) has the following subcommands.

obj define order body

Defines a new order class for an order called order. The new class's fully qualified name will be "obj::order". The body is an oo::define script for the new class; it will define the class's metadata and methods.

The new class will have the superclass specified when the order set was created.

In addition to defining the new class, define does the following:

  • Defines meta name to the be the unqualified order name.
  • Defaults meta title to the order name.
  • Defaults meta sendstates to the empty list.
  • Defaults meta form to the empty string.
  • Defaults meta parmtags to the empty dictionary.
  • Defaults meta monitor to "on".
  • Declares the order(n) instance variable parms() explicitly, so that it can be used without declaration in the class's methods.
  • If the class defines a form, creates the related dynaform(n) and checks it against the order's parameter list.
  • Remembers the order class for later introspection.
obj reset
Resets the object, deleting all defined classes. Note that created dynaforms are not destroyed, as dynaform(n) doesn't support that.
obj names
Returns a list of the unqualified names of the defined orders.
obj exists order
Returns 1 if there's an order called order in the set, and 0 otherwise.
obj validate order
Validates the order name, converting it upper case and returning it, or throwing INVALID if the order isn't in the set.
obj class order
Returns the order's full class name.
obj parms order
Returns a list of the names of the order's parameters.
obj title order
Returns the order's title.
