SYNOPSIS

package require marsgui 3.0.23
namespace import ::marsgui::*
  • multifield pathName ?options...?
  •     -changecmd command
  •     -state state
  •     -table table
  •     -key name
  • pathName cget option
  • pathName configure option value...
  • pathName get
  • pathName set value

DESCRIPTION

The multifield(n) widget is a pseudo-field which adheres to the field(i) interface. It has a value, and set and get methods, but the user cannot directly edit this value.

It is used for dialogs which will update a collection of two or more selected entities; its value is a list of the IDs of the entities. It displays a label that says "num Selected", where num is the number of entries in the list.

COMMANDS

This module defines the following commands.

multifield pathName ?options...?
Creates a new multifield widget named pathName, and returns pathName. The widget supports the following options:
-changecmd command
From field(i). Specifies a command to be called whenever the field's value changes due to explicit calls to set.The new value is appended to the command as an argument.
-state state
From field(i). field(i) widgets have states, just as the normal tk:: and ttk:: widgets do. The valid states are normal and disabled.
-table table
The name of the RDB table with which the multifield is associated. Note that the widget itself doesn't use this value, but it can be queried by clients.
-key name
The name of the key column in the -table from which the IDs are drawn. Note that the widget itself doesn't use this value, but it can be queried by clients.

INSTANCE COMMAND

multifield(n) widgets have the following subcommands:

pathName cget option
From field(i). Returns the value of the specified option.
pathName configure option value...
From field(i). Sets the value of one or more of the object's options.
pathName get
From field(i). Returns the field's current value.
pathName set value
From field(i). Explicitly sets the field's value, and calls the -changecmd if the value has really changed, i.e., successive calls to set with the same value will result in at most one call to the -changecmd.

ENVIRONMENT

marsgui(n) requires Tcl 8.5 or later.

To use marsgui(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
