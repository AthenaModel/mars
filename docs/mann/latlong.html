DESCRIPTION

latlong(n) defines a number of operations on locations expressed as a latitude/longitude pair in decimal degrees. Some of these operations are implemented in C and are available only in the optional Marsbin extension.

COMMANDS

latlong area coords

Computes the area, in square kilometers, of the polygon defined by coords, a flat list of at least three pairs of lat/long coordinates in decimal degrees. Things to note:

  • The polygon must be expressed in counter-clockwise order; if not, the result will be negated.
  • The polygon must be simply connected, with no loops or holes.
  • The polygon may not contain the south or north pole.
  • The edges are assumed to be segments of great circles.
  • Earth is assumed to be a perfect sphere with a diameter of 12472.0 kilometers.
latlong dist loc1 loc2
Computes the spherical distance in kilometers between the two locations.
latlong dist4 lat1 lon1 lat2 lon2
Computes the spherical distance in kilometers between location 1 and location 2.
latlong radius lat lon

Computes the spherical distance in kilometers between location lat,long and some point previously set using latlong pole.

This routine allows radius-related rule checks to be written independent of the location of the pole.

latlong pole ?loc?
Sets/retrieves a location, loc, which is used as the pole for latlong radius computations.
latlong validate loc
Validates location loc, throwing an error if it is not a valid lat/long pair. Otherwise, loc is returned unchanged.
latlong tomgrs loc ?precision?

Binary Extension only. Converts location loc from latitude/longitude to an MGRS coordinate string, colloquially known as a "UTM string". The precision indicates the number of digits to shown for each of easting and northing; it defaults to 3, for 100 meter resolution, but may be set from 0 to 5.

Note that the conversion from lat/long to UTM/MGRS depends on the choice of latlong spheroid.

latlong frommgrs utm

Binary Extension only. Converts an an MGRS coordinate string, colloquially known as a "UTM string", into a pair of latitude/longitude coordinates in decimal degrees.

Note that the conversion from lat/long to UTM/MGRS depends on the choice of latlong spheroid.

Also, note that due to the limited precision of the MGRS format, latlong frommgrs is only an approximate inverse of latlong tomgrs.

latlong togcc loc

Binary Extension only. Converts location loc from latitude/longitude to a set of GCC (Geocentric Coordinate) values. The elevation value at loc is assumed to be zero. The three GCC values returned are in deciman meters.

Note that the conversion from lat/long to GCC depends on the choice of latlong spheroid.

latlong fromgcc xyz

Binary Extension only. Converts a geocentric (GCC) location xyz to a geodetic lat/lon location. The three input GCC values are in deciman meters. The returned lat/lon values are in decimal degreees.

Note that the conversion from GCC to lat/lon depends on the choice of latlong spheroid.

latlong spheroid ?name?

Binary Extension only. Sets/returns the name of the spheroid to use in all coordinate conversions. Defaults to "WE", otherwise known as "WGS 84".

The list of supported spheroids is as follows:

ENVIRONMENT

latlong(n) requires the Marsbin extension for certain operations; the remainder require only Tcl 8.5 or later.

To use latlong(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the marsutil(n) library directory.
