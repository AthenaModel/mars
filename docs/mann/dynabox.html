dynabox(n) -- dynaform(n) Dialog Box, submodule of marsgui(n)

DESCRIPTION

This module defines a tool for popping up modal dialogs containing dynaform(n) forms. The dialogs have these characteristics:

  • OK and Cancel buttons.
  • If desired, a Help button.
  • A user-defined title.
  • dynaform(n)-based content.
  • Optional validation support.

COMMANDS

This module defines the following command:

dynabox popup ?options...?

Pops up a message box containing a text entry widget, given the following options. On "Cancel", returns the empty string; on "OK", validates and returns the form's field/value dictionary. Pressing the window manager's close button is the same as pressing "Cancel".

-resources rdict
Specifies a -resources dictionary to pass along to the dynaview(n) widget, as required by the -formtype.
-formtype ftype
Names the dynaform(n) form type to display in the dialog.
-helpcmd cmd
If given, the dialog will have a Help button; pressing the Help button will call the cmd, which presumably displays related help in some fashion.
-initvalue dict
Specifies an initial value for the form, specified as a dictionary of field names and values. The dict need not contain an entry for every one of the the form's fields.
-oktext text
Specifies the text for the OK button. Defaults to "OK".
-parent window
The message box will appear over the specfied window.
-title string
The dialog window will have the specified title string.
-validatecmd command

Specifies a validation command. The command should take one additional argument, the name/value dictionary to validate.

If the dictionary is valid, the command should return normally; if it returns a text string, the string will be displayed at the bottom of the dialog in black text.

If the dictionary is invalid, the command should do one of two things: First, the command can throw an error with error code INVALID. The error message will be displayed at the bottom of the dialog in red text.

Second, the command can throw an error with error code REJECTED. The error result must then be a dictionary of field names and error messages. The dialog will mark fields with invalid values by coloring their labels red, and when an invalid field is selected its error message will appear at the bottom of the dialog in red text.
