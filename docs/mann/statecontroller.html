  •     -condition expression
  •     -events eventlist
  • menuitem menu itemtype label ?options...?
  • obj cget option
  • obj configure option value...
  • obj control object ?key value...?
  • obj update ?objects?

DESCRIPTION

An instance of the statecontroller(n) is an object that controls the -state of one or more other objects. Controlled objects are typically Tk menu items and buttons; however, statecontroller(n) can control any object with a -state option with values normal and disabled.

The statecontroller(n) updates the state of controlled objects when explicitly asked to do so via its update method. Optionally, it can do so on receipt of one or more notifier(n) events.

The statecontroller(n) determines whether the controlled objects should be enabled or disabled by evaluating a boolean -condition for each. The -condition expression may reference the controlled object as $obj. In addition, each controlled object may specify a dictionary of data items peculiar to that object; this is called the objdict. The -condition may reference any of the entries in the dictionary by name.

COMMANDS

statecontroller name ?options?

Creates a new statecontroller(n) object called name. The object is represented as a new Tcl command in the caller's scope; statecontroller returns the fully-qualified form of the name.

The statecontroller(n) may be created with the following options:

-condition expression

A boolean expression that will be evaluated for each controlled object. The object will be enabled if the expression is true, and disabled if false.

The expression may reference the controlled object as $obj, and any of the values in the object dictionary. See EXAMPLES for details.

-events eventlist
The eventlist must be a flat list of notifier(n) subjects and events. The statecontroller(n) will automatically subscribe to each of the subject/event pairs, and will update all controlled objects whenever one of the events is received. Any event arguments are ignored.
menuitem menu itemtype label ?options...?

Defines a menu item, and returns "$menu $label". This is a convenience function that makes it easier to control menu items; see the EXAMPLES for more. It's equivalent to the following code: 

    $menu add $itemtype -label $label {*}$options

INSTANCE COMMAND

Each instance of the statecontroller object has the following subcommands:

obj cget option
Returns the value of the specified option.
obj configure option value...
Sets the value of one or more of the object's options.
obj control object ?key value...?

Directs the statecontroller to control the -state of the specified object. The remaining arguments collectively form the object's object dictionary.

Usually, the object is the name of a Tk widget instance. Menu items, however, are a special case, as a menu item isn't a widget; rather, it's owned by a widget. If the controlled object is a menu item, then object must be a list of two items: the instance command of the menu on which the item appears, and the index of the item, as defined in for the Tk menu command. The item's label string is a valid index.

obj update ?objects?

By default, this command explicitly updates the -state of all controlled objects. Alternatively, it can update only the listed subset of the controlled objects. This command is useful in two cases:

  • First, when updates need to be triggered by something other than a notifier(n) event, e.g., by a Tk event.
  • Second, when it's known that the triggering event can affect only a small subset of the controlled objects.

EXAMPLES

The following statecontroller is used to enable and disable widgets based on whether the order sent by the widget is valid or not: 

    statecontroller ::cond::orderIsValid -events {
        ::order <State>
    } -condition {
        [::order isvalid $order]
    }

Updates are triggered by receipt of the <State> event from the ::order processor. The controlled widget is enabled if the $order specified in its object dictionary is valid, and disabled otherwise.

A button widget is normally associated with the statecontroller as follows: 

    cond::orderIsValid control \
       [button $win.button                              \
           -text "Create Neighborhood"                  \
           -command [list ::order enter NBHOOD:CREATE]] \
       order NBHOOD:CREATE

Menu items are a special case, because a menu item is an item within a menu widget, not a widget in its own right. Thus, menu items are specified as a list of two items, the menu command and the index of the item, typically the item's label. The menuitem command is a convenience command that creates a menu item and returns exactly such a list: 

    cond::orderIsValid control \
       [menuitem $menu command "Create Neighborhood"    \
           -command [list ::order enter NBHOOD:CREATE]] \
       order NBHOOD:CREATE

ENVIRONMENT

Requires Tcl 8.5 or later.

To use this package in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
