DESCRIPTION

enum(n) defines an enumerated value definition type. Each instance of the type defines an enumeration type: an ordered set of symbolic values. An enum object allows enumerated symbols to be validated, translated to their numeric indices, and vice versa. In addition, each enumerated symbol has two forms: a short symbolic name and a longer descriptive name.

COMMANDS

enum name deflist ?-noindex?

enum defines a new enumeration definition type called name; the short and long names of the enumerated values are defined in the deflist, which has the form: 

short long short long....

For example, 

enum MyEnum {
    G  "The Good"
    B  "The Bad"
    U  "The Ugly"
}

Alternatively, the deflist can be the empty string, "", and values can be added later using the add subcommand.

The name becomes a Tcl command, defined in the current scope; enum returns the fully-qualified form of name.

By default, a valid enumeration value is a short name, a long name, or the numeric index of the short name, where index numbers range from 0 to (size - 1). Allowing index numbers is sometimes convenient, but means that short names and long names can't look like integers. If the -noindex option is included on creation, then numeric indices are no longer valid values; and consequently short names can be numeric.

INSTANCE COMMAND

Each instance of the enum object has the following subcommands:

$enum validate input
Validates that input belongs to the enumeration. If it does, the input is translated to a short name, and returned. If it does not, an error is thrown with an -errorcode of INVALID.
$enum size
Returns the number of symbols in the enumerated type. Given n symbols, the symbols are indexed from 0 to n-1.
$enum names
$enum shortnames
Returns a list of the short symbolic names, in index order. The form shortnames is deprecated.
$enum longnames
Returns a list of the long descriptive names, in index order.
$enum deflist
Returns the enum's current definition list, suitable for creating another enumeration.
$enum index input
Given a short name, a long name, or (if -noindex is omitted) a valid index, returns the index. Given any other input, returns -1. Name lookups are case-insensitive.
$enum name input
$enum shortname input

Given a short name, a long name, or (if -noindex is omitted) a valid index, returns the corresponding short symbolic name. Given any other input, returns "". Name lookups are case-insensitive.

The form shortname is deprecated.

$enum longname input
Given a short name, a long name, or (if -noindex is omitted) a valid index, returns the corresponding long descriptive name. Given any other input, returns "". Name lookups are case-insensitive.
$enum add deflist
Adds additional symbols to the enumeration; this allows the enumeration to be defined a little at a time. The syntax of deflist is defined just as for enum.
$enum eq a b
$enum lt a b
$enum gt a b
$enum le a b
$enum ge a b
These methods compare two enumerated values, a and b, and return 1 if a is equal to, less than, greater than, less than or equal to, or greater than or equal to b, respectively. The values a and b may be short names, long names, or indices; they are converted to indices and compared numerically.
$enum html
Returns an HTML snippet which documents the enumeration's names and long names.

ENVIRONMENT

enum(n) requires Tcl 8.5 or later.

To use enum(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the marsutil(n) library directory.
