DESCRIPTION

range(n) defines a bounded value definition type. Each instance of the type defines a range of numeric values, and can do the following things:

  • Validate that a value belongs to the range.
  • Clamp an arbitrary numeric value within the bounds.
  • Test whether an arbitrary numeric value is within bounds or not.
  • Format values for output.

COMMANDS

range name ?options...?

range defines a new range type called name.

A range can be created with the following options:

-min value
The minimum numeric value for this range. The default is no minimum.
-max value
The maximum numeric value for this range. The default is no maximum.
-format format
A format string used to format numeric values belonging to this range. The default is "%4.2f".

For example, the following code defines "probability" as a numeric value that ranges from 0.0 to 1.0, to be output with three decimal places of precision. 

range probability -min 0.0 -max 1.0 -format {%5.3f}

The name becomes a Tcl command, defined in the current scope; range returns the fully-qualified form of name.

INSTANCE COMMAND

Each instance of the range object has the following subcommands:

$range cget option
Returns the value of the specified option.
$range configure option value...
Sets the value of one or more options.
$range validate input
Validates that input belongs to the range. If it does, it is returned unchanged. If it does not, an error is thrown with error code INVALID.
$range value input
Identical to validate; it is included so that range(n) types can be used interchangeably with quality(n) types in some cases (e.g., mat numerize
$range format value
Given a numeric value of this range type, formats and returns the value as specified by the -format string.
$range inrange value
Returns 1 if the arbitrary value falls within the range's -min and -max, and 0 otherwise.
$range clamp value
Clamps the arbitrary value within the range's -min and -max, and returns the clamped value.

SEE ALSO

enum(n), quality(n).

ENVIRONMENT

range(n) requires Tcl 8.5 or later.

To use range(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the marsutil(n) library directory.
