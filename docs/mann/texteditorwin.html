DESCRIPTION

marsgui(n) contains general-purpose (i.e., non-application-specific) GUI code for use in Mars. This module defines the ::marsgui::texteditorwin, a simple text editor window which can be popped up to edit files in any Tk application.

The text editor has the following features:

  • Multiple edit windows
  • Cut, Copy, Paste, and Undo.
  • New Window, Open File, Save File, Save File As.

texteditorwin(n) is based on the texteditor(n) editing widget.

COMMANDS

This module defines the following command:

texteditorwin pathName ?options...?

Creates a new toplevel texteditorwin window named pathName.

The texteditorwin widget has all the options of the standard Tk "toplevel" widget; in addition, it has the following:

-height height
Sets the height of the editor in lines of text.
-width width
Sets the width of the editor in columns of text.
-background color
Sets the background color of the editor pane.
-foreground color
Sets the foreground color of the editor pane.
-font color
Sets the text font.
-title title
Sets the window title; the name of the current file is added automatically. The title defaults to "Text Editor" appended to the title of the application's main window, ".".
-initialdir dir
Sets the -initialdir passed to tk_getOpenFile and tk_getSaveFile.
-filetypes typelist
Sets the list of file types (-filetypes) passed to tk_getOpenFile and tk_getSaveFile. See the Tk manpages for the format.

INSTANCE COMMAND

win configure option value...
Sets the values of the specified options.
win cget option
Returns the value of the specified option.
win load filename

Loads the named file into the editor. The file will not be loaded if the current file needs to be saved.

The named file need not exist; if it does not, a blank document will be created, but the file name will be saved for when the file is saved.

win open ?filename?
Called with a file name, this subcommand is equivalent to load. Called without a file name, prompts the user to select a file to open. Note that no file can be opened if the current file needs to be saved.

ENVIRONMENT

This package requires Tcl/Tk 8.5 or later.

To use this package in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
