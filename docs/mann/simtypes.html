DESCRIPTION

This module defines data types required by the Mars Simulation Library, simlib(n), and its clients.

QUALITIES

simtypes(n) defines the following quality(n) types.

qaffinity

This quality is used to measure the degree of affinity or relationship between two population groups. Values have the bounds shown in the following table:

Name  Long Name Value Bounds
SUPPORT  Supports  0.8  0.7 < value <= 1.0
LIKE  Likes  0.4  0.2 < value <= 0.7
INDIFF  Is Indifferent To  0.0  -0.2 < value <= 0.2
DISLIKE  Dislikes  -0.4  -0.7 < value <= -0.2
OPPOSE  Opposes  -0.8  -1.0 < value <= -0.7
qcooperation

This quality is used to measure the level of cooperation between two population groups; it can be interpreted as the probability that one group will give information to another group. Values have the bounds shown in the following table:

Name  Long Name Value Bounds
AC  Always Cooperative  100.0  99.9 < value <= 100.0
VC  Very Cooperative  90.0  80.0 < value <= 99.9
C  Cooperative  70.0  60.0 < value <= 80.0
MC  Marginally Cooperative  50.0  40.0 < value <= 60.0
U  Uncooperative  30.0  20.0 < value <= 40.0
VU  Very Uncooperative  10.0  1.0 < value <= 20.0
NC  Never Cooperative  0.0  0.0 < value <= 1.0
qduration

This quality is used to describe the duration of gram(n) level inputs, in decimal days.

Numerically, qduration values have a minimum of 0.0. Symbolically, durations are broken down into these categories:

Name  Long Name Value
XL  X_LONG  5.0
L  LONG  2.5
M  MEDIUM  1.0
S  SHORT  0.1
XS  X_SHORT  0.042
qemphasis

This quality is used to measure an entity's emphasis on agreement or disagreement with respect to a particular topic in a mam(n) belief system. Numerically, the values range from 0.0 to 1.0 and have the bins shown in the following table:

Name  Long Name Value
ASTRONG  Agreement--Strong  0.9
AWEAK  Agreement  0.7
NEITHER  Neither  0.5
DWEAK  Disagreement  0.35
DSTRONG  Disagreement--Strong  0.25
DEXTREME  Disagreement--Extreme  0.15

For this variable, it is more usual to enter data using the symbolic values than to enter specific numbers.

qposition

This quality is used to measure an entity's position for or against a topic in a mam(n) belief system. Numerically, the values range from -1.0 to 1.0, and have the bins shown in the following table:

Name  Long Name Value Bounds
P+  Passionately For  0.9  0.8 < value <= 1.0
S+  Strongly For  0.6  0.45 < value <= 0.8
W+  Weakly For  0.3  0.05 < value <= 0.45
A  Ambivalent  0.0  -0.05 < value <= 0.05
W-  Weakly Against  -0.3  -0.45 < value <= -0.05
S-  Strongly Against  -0.6  -0.8 < value <= -0.45
P-  Passionately Against  -0.9  -1.0 < value <= -0.8
qrel

This quality is used to describe the relationship between two groups.

Numerically, qrel values range from -1.0 to +1.0. Symbolically, relationships are broken down into these categories:

Name  Long Name Value Bounds
FRIEND  Friend  0.5  0.3 < value <= 1.0
NEUTRAL  Neutral  0.1  -0.1 < value <= 0.3
ENEMY  Enemy  -0.5  -1.0 < value <= -0.1
qsaliency

This quality is used to describe the saliency of a concern to a particular pgroup. qsaliency values are used to weight a pgroup's level of satisfaction with each of its concerns. Numerically, qsaliency values range from 0.0 to 1.0. Symbolically, saliency is broken down into these categories:

Name  Long Name Value
CR  Crucial  1.000
VI  Very Important  0.850
I  Important  0.700
LI  Less Important  0.550
UN  Unimportant  0.400
NG  Negligible  0.000
qsat

This quality is used to rate levels of group satisfaction with respect to various concerns. Numerically, qsat values range from -100.0 to +100.0. Symbolically, satisfaction is broken down into these categories:

Name  Long Name Value
VS  Very Satisfied  80.0
S  Satisfied  40.0
A  Ambivalent  0.0
D  Dissatisfied  -40.0
VD  Very Dissatisfied  -80.0
qtrend

This quality is used to describe the trend (slope) of gram(n) satisfaction curves. Symbolically, the possible trends are broken down into these categories:

Name  Long Name Value
VH  Very High  8.0
H  High  4.0
N  Neutral  -1.0
L  Low  -4.0
VL  Very Low  -8.0

ENUMERATIONS

simtypes(n) defines the following enum(n) types.

egrouptype

This enumeration defines the population group types which are used by the simulation.

Index Name Long Name
0 CIV CIVILIAN
1 ORG ORGANIZATION
2 FRC FORCE
eproximity

This enumeration is used to express the proximity of one neighborhood to another.

Index Name Long Name
0 HERE Here
1 NEAR Near
2 FAR Far
3 REMOTE Remote

NUMERIC TYPES

simtypes(n) defines the following numeric types.

ipopulation
This snit::integer defines the set of valid population numbers, integers greater than or equal to zero.
rfraction
This range(n) defines the set of valid proper decimal fractions. Numerically, rfraction values range from 0.0 to 1.0.
rfracpair
This type defines the set of pairs of rfraction's, where the elements of the pair sum to less than or equal to 1.0.
rmagnitude
This range(n) defines the set of valid real magnitudes. Numerically, rmagnitude values range upwards from 0.0.

ENVIRONMENT

This package requires Tcl 8.5 or later.

To use this package in a Tcl script, the environment variable TCLLIBPATH must include the parent of the simlib(n) library directory.
