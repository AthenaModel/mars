DESCRIPTION

order_dialog(n) is responsible for creating and managing dialog boxes for orders defined using order(n).

order_dialog(n)'s dialogs are built around the dynaview(n) widget, which is based on turn on the dynaform(n) form specification included in the order's metadata. See the dynaform(n) man page for information on the standard field types and on how to define custom field types.

COMMANDS

order_dialog(n) provides the following commands:

order_dialog enter options...

Pops up an order dialog given the options. This is the standard way to pop up an order dialog. The options are the same as for the dialog widget, with the addition of -parmdict.

-appname string
Specifies the name of the application, to be used in the title bar of the order dialog.
-flunky flunky
The instance of order_flunky(n) (or a subclass) that is responsible for managing order execution for the dialog.
-helpcmd cmd
If given, the dialog will have a context-sensitive help button. The cmd should be a command prefix taking one additional argument, the name of the order. The command should then make help available for that order.
-master master
Specifies the master window for the dialog, i.e., the application window with which it is associated.
-order name
The name of the order, e.g., "MY:ORDER". The order must be known to the -flunky and must have non-empty form metadata.
-parmdict parmdict
The initial parameter dictionary. It will be used to populate the dialog.
-refreshon events

The value of this option is a list of notifier(n) subjects and events that should cause the dialog to refresh its contents.

The state of an order dialog often depends on the state of the application. As the application state changes, for example, the content of pulldowns might change, thus necessitating a refresh.

order_dialog puck tagdict

This command allows the user to fill in fields in the current order dialog by clicking on elements of the GUI. The tagdict is a dictionary of tag names and values that reflects the entity on which the user clicked. If the current field in the topmost order dialog is tagged with one of the tags in the tagdict, the field will be set to the tag's value.

Field tags are set using the order(n) parmtag metadata value.

INSTANCE COMMAND

Each instance of order_dialog(n) provides the following public methods; however, there should rarely be any need for other modules to use them.

pathName configure option value...
Sets the value of the specified options.
pathName cget option
Returns the value of the specified option.
pathName get
Returns a dictionary of the values of the dialog's fields, by parameter name.
pathName enter parmdict
Sets the values of one or more of the dialog's fields given a dictionary of the parameter names and values. Downstream fields are refreshed as appropriate.

NOTIFIER EVENTS

Each instance of order_dialog(n) sends the following events from subject ::marsgui::order_dialog:

<OrderEntry> tags

An order dialog contains a field for each order parameter. When a given field receives the focus, this event is sent with tags set to the parameter's parmtags as defined in the order's metadata, or "null" if none. When no field has the focus, or the dialog is closed, this event is sent with tags set to the empty string.

This allows GUI components to configure themselves so that the appropriate kind of object can be selected by clicking on the component, e.g., a neighborhood can be selected by clicking on the map.
