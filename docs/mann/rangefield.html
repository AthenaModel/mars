DESCRIPTION

The rangefield(n) widget is a scale(n) configured as a field(i) data entry field. It is used for entering numeric values from a limited range. If the range is specified as a quality(n) type, the field can include a menubox(n) containing the symbolic quality values.

COMMANDS

This module defines the following commands.

rangefield pathName ?options...?

Creates a new rangefield widget named pathName, and returns pathName. The widget supports the following options:

-changecmd command
From field(i). Specifies a command to be called whenever the field's value changes, for any reason whatsoever (including explicit calls to set. The new value is appended to the command as an argument.
-changemode mode

The mode determines how often the rangefield(n) widget calls the -changecmd as the slider is being dragged. If the mode is continuous (the default), then it is called with great frequency. If the mode is onrelease, then it is called only when the user releases the mouse button at the end of the drag.

The onrelease mode is for use when continuous updates are a problem for the application. Note that holding down an arrow key can create the same kind of stream of continual changes. Unfortunately, it also creates a stream of <KeyRelease> events, which makes it impossible to call the -changecmd only when the key is released. Consequently, the slider cannot be adjusted using the keyboard when the mode is onrelease.

-font font
Sets the font for the data label and the menubox(n) of symbolic quality values (if any).
-labelpos position
Determines whether the label is positioned to the left or the right of the slider. The default is right.
-min value
-max value
Creation only. Normally the minimum and maximum bounds for the range come from the -type option; however, sometimes the valid range should be narrower than the type's. Use these options to set a narrower range.
-scalelength pixels
Sets the length of the slider in pixels; the default is 150. Note that if the length is too short, it might not be possible to drag the slider to every legal value.
-state state
From field(i). rangefield(n) widgets have states, just as the normal tk:: and ttk:: widgets do. The valid states are normal and disabled.
-type object

The rangefield(n) must be given a type object with -min and -max options. Mars range(n) and quality(n) objects work, as do snit::integer and snit::double objects (provided that both -min and -max are set).

Note that the user will not be layed out until -type is set; and once -type is set, it cannot be changed.

-resetvalue value
Any field(i) can be set to "" to clear its value. A scale(n) widget has to display some concrete value at all times; this option sets the value to use. If value is "", then the reset value will be 0 if it is in range, and the minimum bound otherwise.
-resolution value

This option specifies the scale's granularity, e.g., if it is set to 1 then the scale's value will be an integer ranging from the minimum value to the maximum value by 1.

If the value is "", then the rangefield(n) will pick a resolution automatically.

-showreset flag
If true, the field will include a "Reset" button at the right end. Pressing the button will reset the field to its -resetvalue.
-showsymbols flag
If true, the -type is assumed to be a quality(n) object. The object's long symbolic names will appear in a pulldown at the right end of the field. The user can select a particular symbol and get the matching value. If the user sets the value using the slider, the pulldown will show the symbol that most closely matches the selected value.

INSTANCE COMMAND

rangefield(n) widgets have the following subcommands:

pathName cget option
From field(i). Returns the value of the specified option.
pathName configure option value...
From field(i). Sets the value of one or more of the object's options.
pathName get
From field(i). Returns the field's current value.
pathName set value

From field(i). Explicitly sets the field's value, and calls the -changecmd if the value has really changed, i.e., successive calls to set with the same value will result in at most one call to the -changecmd.

If the value is the empty string, the field will display no value, the scale will be reset to the -resetvalue, and the -showsymbols pulldown, if any, will be blank.

ENVIRONMENT

marsgui(n) requires Tcl 8.5 or later.

To use marsgui(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
