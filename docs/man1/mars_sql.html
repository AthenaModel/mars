DESCRIPTION

mars sql db_file [-create]

mars_sql(1) is a browser for SQLite database files. It allows the user to view the database schema and enter arbitrary queries, including updates and deletes.

When invoked with the name of an SQLite database file, mars_sql(1) opens the file using an sqldocument(n) object, and pops up a command-line interface. Any SQL statement may be entered and executed. Statements can extend onto multiple lines, and must end with a semi-colon. The dialect of SQL is SQL-92 as implemented by SQLite; sqldocument(n) adds a number of new SQL functions.

If the -create option is specified, the named SQLite database file will be created. If -create is used with a filename that already exists, the command will be rejected.

If the name of the database is the special name ":memory:" then a new database is created in memory. In this case the -create option has no meaning.

In addition to SQL statements, mars_sql(1) also accepts a number of special commands; see COMMANDS for details.

The result of SELECT queries will be displayed in either Multicolumn or List mode. Multicolumn mode displays the results in tabular form, with one line of text per record and field names at the top of each column; long values are truncated. List mode displays each record individually, with field labels and values in two parallel columns. The default is Multicolumn mode; the mode may be changed from the View menu or via the .mode command.

By default, all entered queries are executed. If "explain" mode is enabled from the View menu or the toolbar, SQLite's "EXPLAIN QUERY PLAN" statement will be used to retrieve information about the tables and indexes used.

MENUS

mars_sql(1) provides the following menus:

File Menu

Exit
Terminates the application.

Edit Menu

Cut
Cuts selected text to the clipboard.
Copy
Copies selected text to the clipboard.
Paste
Pastes the text from the clipboard.
Select All
Selects all text in the current widget.

View Menu

Multicolumn Mode
Selects multicolumn output mode (.mode mc).
List Mode
Selects list output mode (.mode list).
Execute Mode
Executes SQL queries as usual.
Explain Mode
Outputs a description of the tables and indices used by a query without actually executing it. (This is equivalent to beginning the query with the words "EXPLAIN QUERY PLAN".)

COMMANDS

By default, all command input is assumed to be SQL. mars_sql(1) also supports a number of special commands, all of which begin with ".". Special commands have Tcl syntax.

.mode ?newMode?
By default, returns the current output mode, mc (multicolumn) or list. If newMode is given, sets the mode accordingly.
.schema ?table?
By default, returns the SQL CREATE statements used to define all tables, views, and indices in the database. If given, table may the name of a specific table, view, or index; or a glob pattern. Either way, the output will include only the matching tables, views, and indices.
.tables
Returns a list of the names of the tables defined in the database.
