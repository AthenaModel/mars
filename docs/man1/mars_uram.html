DESCRIPTION

mars uram ?options...? ?database?

mars_uram(1) is a tool for working with uram(n) interactively, and especially for performance optimization. It loads URAM data from a uramdb(5) file into an instance of uram(n), and allows the user to schedule attitude inputs, query inputs and outputs, and advance simulation time.

The uramdb(5) file name may be specified on the command line, in which case it will be loaded automatically. The database file will be loaded before any -script specified on the command-line is executed. Alternatively, uramdb(5) files can be loaded interactively using the load command.

The mars_uram(1) user interface consists of three areas: a simulation tool bar, a browser area containing the scrolling log and (ultimately) a number of data browsers, and an interactive command-line interface (CLI). As the model is exercised, a trace of activities is written to the scrolling log. The CLI provides a terminal-like command line interface at which mars_uram(1) commands can be typed; for example, entering the command select will dump the results of a satisfaction query at the CLI.

Note that the boundary between the browser area and the CLI can be dragged up or down with the mouse.

See MENUS for a description of mars_uram(1)'s menus. See COMMANDS for a full list of mars_uram(1) commands. See CLI for details on the shell's editing features. See SCRIPTS for details on how to write mars_uram(1) scripts.

Simulation Time

mars_uram(1) measures simulation time in timesteps called "ticks", which are nominally one week long. However, mars_uram(1) has no notion of simulation time beyond the tick.

OPTIONS

mars_uram(1) may be invoked with the following command-line options:

-help
Writes the command-line syntax to standard output, and halts.
-script scriptFile
The specified script file will executed automatically.

MENUS

mars_uram(1) provides the following menus:

File Menu

New Browser...
Opens a new window containing a set of browser tabs.
Load uramdb(5) File...
Pops up the "Load uramdb(5) File..." dialog. The selected file will be loaded into the simulation.
Save RDB File...
Saves the current contents of the RDB, including all uramdb(5) and uram(n) data, to disk as an RDB file.
Save CLI Scrollback Buffer...
Saves the current contents of the CLI's scrollback buffer to disk as a text file.
Exit
Terminates the application.

Edit Menu

Cut
Cuts selected text to the clipboard.
Copy
Copies selected text to the clipboard.
Paste
Pastes the text from the clipboard.
Select All
Selects all text in the scrolling log or CLI.

COMMANDS

mars_uram(1) provides the following commands, which may be entered at the CLI or used in SCRIPTS:

= expression...

Computes and returns the value of an arithmetic expression: 

    > = 1 + 1
    2
    >

Variables can be defined with set and retrieved using "$" notation: 

    > set num 5
    5
    > = $num * 2
    10
    >
bgerrtrace
Returns (and logs) the stack trace from the most recent bgerror.
call script
Calls the named script, which is assumed to be in the current working directory unless a path is specified. The ".tcl" extension may be omitted; call will add it automatically.
coop subcommand args...

This command allows the user to enter cooperation inputs. The subcommands are as follows:

coop badjust driver f g delta
coop bset driver f g value
coop cset f g value...
coop persistent driver cause f g mag ?options...?
coop transient driver cause f g mag ?options...?
These subcommands are delegated to uram(n) as is.
coop mass ?driver?
This subcommand creates cooperation inputs en masse, with -p and -q both set to 1.0. The inputs are associated with the specified driver; if no driver is specified, one is generated automatically.
debug ?command...?

Invokes a debugger window. The debugger is simply a Tcl interpreter with full access to the application's code and data; it's intended for developer use only.

If a command name is given, the command will be loaded into the debugger's mod editor.

errtrace
When used interactively, returns the stack trace associated with the last command which returned an error. This command is a debugging aid, and has no effect on the execution of the program.
help ?-info? command...

Returns the calling syntax for the named command in human-readable format. If -info is specified, then information on the linkage from the executive commands to the simulation internals is included in the output.

For example, 

> help rdb
Usage: rdb subcommand ?args...?
Valid subcommands: eval, query, schema, table
> help rdb query
Usage: rdb query sql
>
hrel subcommand args...

This command allows the user to enter horizontal relationship inputs. The subcommands are as follows:

hrel badjust driver f g delta
hrel bset driver f g value
hrel cset f g value...
hrel persistent driver cause f g mag
hrel transient driver cause f g mag
These subcommands are delegated to uram(n) as is.
hrel mass ?driver?
This subcommand creates horizontal relationship inputs en masse. The inputs are associated with the specified driver; if no driver is specified, one is generated automatically.
load filename
Tries to load the named uramdb(5) file. mars_uram(1) looks for the file in the current working directory, which is the location from which mars_uram(1) was invoked; alternatively, you can specify an absolute path.
loadperf ?options...?

Creates a synthetic scenario for performance testing, and loads it into uram(n). See uramdb(n) mkperfdb for the options.

The call to uram(n) init is profiled automatically.

log message

Writes message to the scrolling log, e.g., 

    100000ZJAN05> log "Hello, World!"
now
Returns the current simulation time in ticks.
parm subcommand args...

The parm command sets and queries preference parameters. Note that parameter values are saved automatically to ~/.mars_uram/parmdb.defaults when the program exits, and reloaded at start up.

parm get parm
Retrieves the value of the named parameter.
parm help parm
Retrieves the documentation string for the parameter.
parm list ?pattern?
Returns a human-readable list of parameters whose names match the specified wildcard pattern, in alphabetical order. If no pattern is specified, all parameters will be returned.
parm load
Loads the parameter set from "~/.mars_uram/parmdb.defaults"; this also happens automatically at start up.
parm names ?pattern?
Returns a list of the parameter names, in alphabetical order, which match the specified wildcard pattern; if no pattern is specified, all names will be returned.
parm set parm value
Sets the value of the named parameter, which must be valid.
parm reset
Resets all parameters to their default values.
parm save
Saves the parameter set to "~/.mars_uram/parmdb.defaults"; this also happens automatically at shut down.
profile command ?args...?
Executes the command and times its execution. The time in microseconds and the command are logged, and the command's result is returned.
rdb subcommand ?args...?

Queries the application's run-time database. The subcommands are as follows:

rdb eval sql

Executes the specified sql query and returns a list of the results. Note that the query will not be allowed to modify the database.

This command is appropriate for use in scripts; prefer rdb query or select for interactive use.

rdb query sql
Executes the specified sql query and returns the results in tabular form. See also select, which is often more convenient.
rdb schema ?table?
Returns the SQL schema for the named table, or by default the entire database.
rdb tables
Returns a list of the names of the database tables.
reset
Re-initializes URAM back to simulation time 0; this is equivalent to reloading the uramdb(5) file.
sat subcommand args...

This command allows the user to enter satisfaction inputs. The subcommands are as follows:

sat badjust driver g c delta
sat bset driver g c value
sat cset g c value...
sat persistent driver cause g c mag ?options...?
sat transient driver cause g c mag ?options...?
These subcommands are delegated to uram(n) as is.
sat mass ?driver?
This subcommand creates satisfaction inputs en masse, with -p and -q both set to 1.0. The inputs are associated with the specified driver; if no driver is specified, one is generated automatically.
select sql...

Executes the SQL query and returns the results in tabular form. This is simply a wrapper around rdb query; compare 

  > select * from uramdb_n

with 

  > rdb query {select * from uramdb_n}
set variable value
Assigns a value to a named variable.
step ?ticks?

Advances simulation time by the specified number of ticks, if given; otherwise, by 1 tick.

The call to uram(n) advance is profiled automatically.

super args...
Concatenates its arguments into a single command, and executes that command in the context of the application rather than in the URAM Workbench executive.
unload
Unloads the current uramdb(5) file.
vrel subcommand args...

This command allows the user to enter vertical relationship inputs. The subcommands are as follows:

vrel badjust driver g a delta
vrel bset driver g a value
vrel cset g a value...
vrel persistent driver cause g a mag
vrel transient driver cause g a mag
These subcommands are delegated to uram(n) as is.
vrel mass ?driver?
This subcommand creates vertical relationship inputs en masse. The inputs are associated with the specified driver; if no driver is specified, one is generated automatically.

In addition to the specific commands listed above, mars_uram(1) supports most standard Tcl commands.

CLI

The mars_uram(1) command-line interface (CLI) provides the following conveniences:

  • Full command-line editing. Use the left and right arrow keys (or the mouse) to edit the text on the command-line before executing it.
  • Command-history. Use the up and down arrow keys to retrieve the previous commands to the command-line for execution.
  • Command-completion. Press the Tab key to expand a partial command name into a full command name, e.g., type "st" and press Tab to expand "st" into "step".
  • Standard cut, copy, and paste. Use Ctrl+X, Ctrl+C, and Ctrl+V to cut, copy, and paste on the command line. Use the mouse to select older text in the shell (e.g., command outputs) and copy it with Ctrl+C.

SCRIPTS

A mars_uram(1) script is simply a sequence of COMMANDS placed on consecutive lines of a text file, e.g., 

    load mydata.uramdb
    step

Use the call command to execute a script: 

    > call myscript.tcl

mars_uram(1) scripts are in fact Tcl scripts, and can define procs, perform loops, and so forth.
